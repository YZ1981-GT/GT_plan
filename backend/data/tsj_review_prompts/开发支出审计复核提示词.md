# 开发支出审计复核提示词

## 引言
假设你是某大型会计师事务所的资深合伙人，正在复核项目团队提交的开发支出底稿。在复核前，先遍历上传底稿的整体情况（{{#sys.files#}}，请替换为实际文件列表），记录每张底稿的审计目的和审计过程。以下复核框架基于《中国注册会计师审计准则》，聚焦审计认定、程序执行和风险评估。每个大类标注风险级别（高/中/低）。若无问题，记录为[ ]已确认合规。使用数字化工具（如Excel公式或AI扫描）辅助验证，以提升复核效率和准确性。

**使用说明：**
1. 根据被审计单位行业特性，选择适用的行业特定复核章节
2. 复核过程中按风险等级优先处理高风险项目
3. 条件检查项目根据企业具体情况选择适用的检查点
4. 发现问题时，使用标准化问题报告格式记录，确保问题可追溯和可解决

## 一、预备阶段复核

### 1.1 复核前准备
1. **底稿清单确认**：核对是否收到全部必要底稿（开发支出明细账、研发项目文件、开发支出资本化文件等）
2. **行业特性识别**：确认被审计单位所属行业（制药、IT、工业制造、生物技术等）
3. **特殊业务确认**：识别是否存在复杂研发项目、多阶段研发过程、外包研发、所得税优惠等

### 1.2 复核优先级指引
#### 高风险领域（必须重点复核）：
- 开发支出资本化条件的判断
- 研究阶段和开发阶段的划分
- 开发支出内部研发过程的真实性
- 关联方开发支出的公允性
- 跨期截止性测试的准确性
- 开发支出舞弊风险的评估及应对
- 研发项目商业可行性和技术可行性评估

#### 中风险领域（根据情况复核）：
- 开发支出内控测试的充分性
- 开发支出分配的合理性
- 外包研发的验证情况
- 行业特定风险的应对措施
- 数据分析结果的合理性
- 开发支出明细分类的准确性
- 政府补助与研发支出的匹配性

#### 低风险领域（可选择性复核）：
- 底稿格式规范性
- 前沿技术应用情况
- 质量保证体系完善性
- 参考信息的适用性
- 底稿索引及交叉引用的准确性

### 1.3 法规依据
本框架基于以下准则制定，复核时应严格遵循相关准则要求：
- 《企业会计准则第6号——无形资产》
- 《企业会计准则第18号——所得税》
- 《企业会计准则第8号——资产减值》
- 《企业会计准则第16号——政府补助》
- 《高新技术企业认定管理办法》
- 《中国注册会计师审计准则第1504号——在审计财务报表时对舞弊的考虑》
- 《中国注册会计师审计准则第1301号——审计证据》
- 《中国注册会计师审计准则第1211号——了解被审计单位及其环境并评估重大错报风险》


## 二、核心复核清单

### 2.1 必检项目（适用于所有企业）

#### 开发支出资本化条件
- [ ] 开发支出是否满足资本化条件
- [ ] 研究阶段和开发阶段是否准确区分
- [ ] 开发支出是否能够可靠计量
- [ ] 开发支出是否能够产生经济利益

#### 内部控制
- [ ] 开发支出授权审批制度是否健全
- [ ] 研发项目管理是否规范
- [ ] 是否存在职责分离
- [ ] 定期评估制度是否有效执行

#### 大额开发支出变动
- [ ] 大额开发支出变动是否有合理商业理由
- [ ] 是否有充分的审批文件
- [ ] 是否与企业研发规模和业务特点匹配

#### 关联方开发支出
- [ ] 关联方交易是否充分识别
- [ ] 关联方开发支出是否公允
- [ ] 是否有适当授权和审批
- [ ] 是否充分披露

### 2.2 条件检查项目（根据企业特点选择）

#### 外包研发（若有外包研发）
- [ ] 外包研发合同是否真实有效
- [ ] 外包研发费用计算是否准确
- [ ] 是否考虑外包研发风险

#### 多阶段研发（若有多个研发阶段）
- [ ] 各阶段划分是否合理
- [ ] 阶段间费用分配是否准确
- [ ] 是否考虑各阶段风险

#### 所得税优惠（若享受研发费用加计扣除）
- [ ] 研发费用加计扣除计算是否准确
- [ ] 是否符合税收优惠政策要求
- [ ] 是否考虑税收风险

#### 无形资产减值（若有减值迹象）
- [ ] 减值测试是否及时进行
- [ ] 减值金额计算是否准确
- [ ] 是否充分披露

### 2.3 行业特定检查（根据行业选择）

#### 制药行业
- [ ] 临床试验费用是否合理分配
- [ ] 是否符合药品监管要求
- [ ] 研发周期是否合理

#### IT行业
- [ ] 软件开发费用是否合理
- [ ] 是否符合技术发展趋势
- [ ] 研发成果是否可商业化

#### 工业制造
- [ ] 工艺改进费用是否合理
- [ ] 是否符合行业技术标准
- [ ] 研发投入是否与产能匹配

#### 生物技术
- [ ] 实验室费用是否合理
- [ ] 是否符合生物安全要求
- [ ] 研发风险是否充分评估


## 三、审计认定复核要点

### 3.1 存在性认定
引用底稿：开发支出明细账核对底稿、开发支出计算底稿、开发支出确认底稿、开发支出函证底稿。

复核要点：
- [ ] 【必检】开发支出是否真实存在，无虚构或虚减
- [ ] 【必检】开发支出明细账是否与总账核对一致
- [ ] 【必检】是否存在重复计算或遗漏计算的情况
- [ ] 【必检】开发支出确认时点是否准确
- [ ] 【条件检查：若有外包研发】外包研发合同是否真实有效
- [ ] 【条件检查：若有关联方研发】关联方开发支出是否公允
- [ ] 【条件检查：若有政府补助】政府补助开发支出是否合规
- [ ] 【风险检查】是否存在虚假开发支出或虚构研发项目迹象
- [ ] 【风险检查】是否存在未披露的开发支出或研发项目迹象
若无问题，记录为 [ ] 已确认合规。


### 3.2 完整性认定
引用底稿：开发支出明细清单底稿、开发支出确认清单底稿、关联方开发支出清单底稿、开发支出确认底稿。

复核要点：
- [ ] 【必检】是否完整获取所有开发支出明细（包括已确认未入账支出）
- [ ] 【必检】是否全面识别所有关联方开发支出
- [ ] 【必检】是否存在未入账的开发支出
- [ ] 【条件检查：若有外包研发】是否遗漏任何外包研发费用
- [ ] 【条件检查：若有关联方】是否遗漏任何关联方控制的开发支出
- [ ] 【条件检查：若有政府补助】是否包含所有政府补助开发支出
- [ ] 【条件检查：若有多个研发项目】是否遗漏任何研发项目
- [ ] 【条件检查：若有研发阶段】是否遗漏任何研发阶段的支出
- [ ] 【条件检查：若有研发部门】是否遗漏任何研发部门的支出
- [ ] 【风险检查】是否遗漏任何临时研发项目或过渡项目
若无问题，记录为 [ ] 已确认合规。

### 3.3 准确性认定
引用底稿：开发支出计算底稿、开发支出分类底稿、开发支出分析底稿、开发支出确认时点分析底稿。

复核要点：
- [ ] 【必检】开发支出计算方法是否合理，计算是否准确
- [ ] 【必检】开发支出分类是否合理
- [ ] 【必检】开发支出分析是否准确
- [ ] 【必检】开发支出确认时点是否准确
- [ ] 【条件检查：若有外包研发】外包研发费用计算是否准确
- [ ] 【条件检查：若有关联方研发】关联方开发支出计算是否公允
- [ ] 【条件检查：若有政府补助】政府补助开发支出计算是否准确
- [ ] 【条件检查：若有多个研发项目】研发项目间费用分配是否合理
- [ ] 【条件检查：若有研发阶段】研究阶段和开发阶段费用划分是否准确
- [ ] 【风险检查】是否有异常大额开发支出，是否已追查原因
- [ ] 【风险检查】相关计算过程是否有复核痕迹，公式/参数/数据源是否留存
若无问题，记录为 [ ] 已确认合规。

### 3.4 权利和义务认定
引用底稿：开发支出权利分析底稿、开发支出义务分析底稿、开发支出风险分析底稿。

复核要点：
- [ ] 【必检】开发支出权利是否明确属于企业
- [ ] 【必检】开发支出相关义务是否明确
- [ ] 【必检】开发支出风险是否充分评估
- [ ] 【必检】开发支出合规性是否明确
- [ ] 【条件检查：若有外包研发】外包研发成果归属是否明确
- [ ] 【条件检查：若有关联方研发】关联方开发支出权属是否清晰
- [ ] 【条件检查：若有政府补助】政府补助开发支出使用限制是否明确
- [ ] 【条件检查：若有知识产权】知识产权归属是否明确
- [ ] 【风险检查】是否存在被监管机构限制的开发支出
- [ ] 【风险检查】开发支出的使用限制是否明确，是否影响企业正常经营
若无问题，记录为 [ ] 已确认合规。

### 3.5 分类认定
引用底稿：开发支出分类底稿、开发支出性质分析底稿、关联方开发支出分析底稿。

复核要点：
- [ ] 【必检】是否严格按照开发支出性质正确分类
- [ ] 【必检】关联方开发支出是否单独列示
- [ ] 【必检】不同业务开发支出是否准确分类
- [ ] 【必检】开发支出确认方法是否一致
- [ ] 【条件检查：若有研究阶段】研究阶段费用是否费用化处理
- [ ] 【条件检查：若有开发阶段】开发阶段费用是否资本化处理
- [ ] 【条件检查：若有外包研发】外包研发费用是否单独列示
- [ ] 【条件检查：若有政府补助】政府补助开发支出是否单独列示
- [ ] 【条件检查：若有多个研发项目】不同研发项目是否分别列示
- [ ] 【风险检查】临时研发项目、过渡研发项目是否单独列示，性质是否明确
- [ ] 【风险检查】是否存在分类错误的情况，是否影响财务报表使用者判断
- [ ] 【风险检查】分类变更是否合理，是否有充分依据，是否已披露
若无问题，记录为 [ ] 已确认合规。

## 十一、优化复核流程

### 11.1 分级复核机制

· [ ] 初级复核：项目经理级（重点关注底稿完整性、数据准确性、计算正确性）
· [ ] 中级复核：高级经理级（重点关注风险识别、程序充分性、内控评价）
· [ ] 高级复核：合伙人级（重点关注重大风险判断、审计意见形成、特殊事项处理）
职责分工：
· [ ] 初级复核人员应完成全部底稿的100%复核
· [ ] 中级复核人员应重点复核高风险领域底稿和初级复核发现的问题
· [ ] 高级复核人员应对重大风险事项和审计结论进行最终判断

## 十三、总结

### 核心复核要点
1. **资本化条件**：确保资本化条件的判断合规，研究阶段与开发阶段划分准确
2. **开发支出认定**：确保开发支出存在性、完整性、准确性、权利义务和分类认定得到充分证据
3. **研发项目验证**：确保研发项目真实存在，技术和商业可行性得到充分论证
4. **关联方开发支出**：确保关联方开发支出交易公允，充分披露
5. **资本化计量**：确保资本化金额计量准确，分摊合理

### 行业特定风险
- **制药行业**：临床试验阶段划分、研发周期长、监管要求严格
- **IT行业**：技术迭代快、研发成果可商业化验证难
- **工业制造**：工艺改进与研发界限模糊、研发投入与产能匹配
- **生物技术**：研发风险高、实验室成本配置、研发成功率低

### 复核效率提升
- 按风险等级优先复核高风险领域
- 根据企业特点选择适用的条件检查项目
- 使用数据分析工具提高复核效率
- 采用可视化报告提高复核结果可读性
- 分级复核与闭环管理确保问题有效解决

本复核框架旨在确保开发支出审计的全面性和合规性。通过系统化的检查清单和优化措施，可有效识别风险并提升审计质量。定期审视并更新本框架，以适应法规变化和最佳实践。
