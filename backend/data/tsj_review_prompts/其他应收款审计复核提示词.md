# 其他应收款审计复核提示词

## 引言
假设你是某大型会计师事务所的资深合伙人，正在复核项目团队提交的其他应收款底稿。在复核前，先遍历上传底稿的整体情况（{{#sys.files#}}，请替换为实际文件列表），记录每张底稿的审计目的和审计过程。以下复核框架基于《中国注册会计师审计准则》，聚焦审计认定、程序执行和风险评估。每个大类标注风险级别（高/中/低）。若无问题，记录为[ ]已确认合规。使用数字化工具（如Excel公式或AI扫描）辅助验证，以提升复核效率和准确性。

## 复核框架

### 一、审计目标明确性复核（高风险，参照《中国注册会计师审计准则》第1301号）
#### 1.1 存在性认定
引用底稿：其他应收款函证相关底稿、其他应收款明细账核对底稿、相关合同协议底稿。
复核要点：
· [ ] 函证是否全面覆盖重要债务人，回函是否直接寄回事务所
· [ ] 其他应收款明细账是否与总账核对一致
· [ ] 是否存在虚构其他应收款或虚增债权的情况
若无问题，记录为 [ ] 已确认合规。

#### 1.2 完整性认定
引用底稿：其他应收款明细清单底稿、相关合同协议清单底稿、关联方交易清单底稿。
复核要点：
· [ ] 是否完整获取所有其他应收款明细（包括已核销账户）
· [ ] 是否全面识别所有关联方其他应收款
· [ ] 是否存在未入账的其他应收款交易

#### 1.3 准确性认定

· [ ] 其他应收款余额计算是否准确

## 九、优化复核流程
9.1 分级复核机制
复核级别：
· [ ] 初级复核：项目经理级（重点关注底稿完整性、数据准确性、计算正确性）
· [ ] 中级复核：高级经理级（重点关注风险识别、程序充分性、内控评价）
· [ ] 高级复核：合伙人级（重点关注重大风险判断、审计意见形成、特殊事项处理）
职责分工：
· [ ] 初级复核人员应完成全部底稿的100%复核
· [ ] 中级复核人员应重点复核高风险领域底稿和初级复核发现的问题
· [ ] 高级复核人员应对重大风险事项和审计结论进行最终判断
若无问题，记录为 [ ] 已确认合规。

9.2 复核时间节点和进度跟踪
时间节点：
· [ ] 现场审计阶段：每周五完成当周底稿初级复核
· [ ] 现场结束后：3个工作日内完成全部底稿初级复核
· [ ] 报告出具前：7个工作日内完成中级和高级复核
进度跟踪：
· [ ] 建立复核进度跟踪表，记录各底稿复核状态（未复核/复核中/已复核/问题整改中）
· [ ] 每周进行复核进度例会，更新复核状态
· [ ] 对复核滞后事项进行风险提示和资源调配
若无问题，记录为 [ ] 已确认合规。

9.3 复核问题闭环管理
问题跟踪：
· [ ] 建立问题清单，记录问题编号、类型、严重程度、责任人、解决期限
· [ ] 设置问题状态（已发现/已分配/解决中/已解决/已验证）
· [ ] 对超期未解决问题进行升级处理
问题验证：
· [ ] 所有问题必须由提出人验证解决方案的有效性
· [ ] 重大问题需由更高级别复核人员参与验证
· [ ] 形成问题-解决方案知识库，用于后续项目参考
若无问题，记录为 [ ] 已确认合规。

## 十、提升报告格式实用性
10.1 问题严重程度分级标准
严重程度分级：
· [ ] 一级问题（高风险）：直接影响审计意见、涉及重大错报、违反法律法规
  示例：未发现虚构其他应收款、函证程序存在重大缺陷
· [ ] 二级问题（中风险）：可能影响特定审计认定、需要调整的重要事项
  示例：重要其他应收款未得到合理解释、坏账准备计提存在系统性错误
· [ ] 三级问题（低风险）：不影响审计结论但需要改进的事项
  示例：底稿引用不准确、格式不规范、描述不清晰
问题上报流程：
· [ ] 一级问题：立即上报项目合伙人和质量复核合伙人
· [ ] 二级问题：24小时内上报项目经理和高级经理
· [ ] 三级问题：正常提交项目组整改
若无问题，记录为 [ ] 已确认合规。

格式要求：

· [ ] 异常指标应有醒目标识和简要解释
