# C11 管理业务循环控制测试底稿模板库

## 📖 文档概述

本文档为管理业务循环控制测试底稿模板库，包含管理相关的所有控制测试程序模板，适用于致同会计师事务所的审计工作。

## 🚀 使用说明

1. **控制测试选择**：根据被审计单位的实际情况选择合适的控制测试模板
2. **程序执行**：按照底稿中的控制测试程序逐步执行
3. **记录完整**：确保所有控制测试程序都有完整的执行记录
4. **偏差处理**：及时识别和处理控制偏差

## 📋 标准化选项清单

### 🎯 认定维度选项
- **存在/发生**：确认管理业务交易和事项确实发生，管理费用确实存在
- **完整性**：确认所有应当记录的管理业务交易和事项均已记录
- **准确性、计价和分摊**：确认管理业务交易和事项以恰当的金额记录
- **权利和义务**：确认企业拥有或控制管理业务相关的权利和义务
- **截止**：确认管理业务交易和事项记录于正确的会计期间
- **分类**：确认管理业务交易和事项记录于恰当的账户
- **列报**：确认管理业务在财务报表中得到恰当列报和披露

### ⚙️ 控制属性选项
- **人工的**：完全由人工执行的控制活动
- **自动化的**：由信息系统自动执行的控制活动
- **人工依赖信息系统控制**：依赖信息系统产生信息的人工控制

### ⏰ 频率选项
- **每笔交易**：针对每笔管理业务交易执行的控制
- **每天**：每个工作日执行一次的控制
- **每周**：每周执行一次的控制
- **每半月**：每半个月执行一次的控制
- **每月**：每月执行一次的控制
- **每季度**：每季度执行一次的控制
- **每年**：每年执行一次的控制
- **非常规/低运行频率**：不定期或根据特定情况执行的控制
- **其他（描述）**：其他特定频率的控制

### 🎚️ 与控制相关的风险等级
- **高**：控制失效可能导致重大错报风险
- **中**：控制失效可能导致中等程度错报风险
- **低**：控制失效导致的错报风险相对较低

### 🔍 控制测试方法详细说明

#### 1. 询问
**定义**：是指向被审计单位相关人员进行询问
**适用情况**：了解控制设计和执行情况
**执行要点**：
- 选择合适的询问对象
- 准备结构化的询问提纲
- 获取书面确认（如需要）

#### 2. 检查
**定义**：是指对被审计单位相关文件或记录进行检查
**适用情况**：验证控制执行的书面证据
**执行要点**：
- 检查原始凭证和记录
- 关注审批签字和日期
- 验证文件的完整性和真实性

#### 3. 观察
**定义**：是指察看相关人员执行某项控制活动的过程
**适用情况**：了解控制的实际执行过程
**执行要点**：
- 在正常业务环境下观察
- 记录观察到的关键控制步骤
- 注意控制执行的一致性

#### 4. 重新执行
**定义**：是指独立执行作为被审计单位内部控制组成部分的程序或控制
**适用情况**：验证控制的有效性
**执行要点**：
- 使用相同的数据和参数
- 比较执行结果的一致性
- 分析差异原因

#### 5. 前期
**定义**：利用以前审计获取的某些控制测试的审计证据
**适用情况**：控制未发生变化的情况
**执行要点**：
- 评估控制变化情况
- 确定可依赖的程度
- 补充必要的当期测试

#### 6. 前推
**定义**：利用期中已获取的控制测试证据推断至期末
**适用情况**：期中测试基础上的期末推断
**执行要点**：
- 评估剩余期间的控制变化
- 执行补充测试程序
- 考虑控制环境变化

#### 7. 利用内部审计工作
**定义**：在某些情况下，注册会计师可以利用内部审计人员的工作
**适用情况**：内部审计工作质量可靠的情况
**执行要点**：
- 评估内部审计的客观性和胜任能力
- 复核内部审计工作质量
- 确定利用程度

#### 8. 利用服务机构的审计师的报告
**定义**：服务注册会计师对服务机构控制的报告
**适用情况**：使用服务机构处理管理业务的情况
**执行要点**：
- 评估报告的相关性和可靠性
- 关注报告涵盖的期间
- 考虑补充控制的必要性

## 📚 标准化选项使用指南

### 管理业务循环常用组合
1. **费用审批控制**：存在/发生 + 准确性、计价和分摊
2. **预算管理控制**：完整性 + 准确性、计价和分摊
3. **资产管理控制**：存在/发生 + 权利和义务
4. **合同管理控制**：权利和义务 + 列报

### 选择建议
- **重大管理费用**：建议选择"高"风险等级，重点关注存在/发生认定
- **预算控制**：重点测试"完整性"和"准确性、计价和分摊"认定
- **资产管理**：重点关注"存在/发生"和"权利和义务"认定

## 📊 管理循环业务流程图及描述

### 管理循环主要业务流程

```
费用需求 → 费用申请 → 费用审批 → 预算控制 → 费用执行 → 费用审核 → 费用报销 → 合同管理 → 重大决策 → 档案管理 → 合规审查
    ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓
  各部门     申请部门    财务部     财务部     执行部门    财务部     财务部     法务部     董事会     档案部     合规部
```

### 详细流程描述

**A. 费用管理阶段**
1. **费用需求识别**：各部门识别费用支出需求
2. **费用申请提交**：按照规定提交费用申请
3. **费用审批决策**：按照审批权限进行费用审批
4. **费用执行监控**：监控费用执行情况
5. **费用报销处理**：处理费用报销事务

**B. 预算管理阶段**
1. **预算编制**：编制年度和月度预算
2. **预算审批**：审批预算方案
3. **预算执行**：按照预算执行费用支出
4. **预算监控**：监控预算执行情况
5. **预算调整**：根据需要调整预算

**C. 合同管理阶段**
1. **合同需求**：识别合同签订需求
2. **合同起草**：起草合同条款
3. **合同审核**：审核合同内容
4. **合同签订**：正式签订合同
5. **合同执行**：监督合同执行

**D. 决策管理阶段**
1. **决策需求**：识别重大决策需求
2. **决策准备**：准备决策相关资料
3. **决策审议**：组织决策审议
4. **决策执行**：执行决策方案
5. **决策监控**：监控决策执行效果

### 关键控制点识别

| 业务环节 | 关键风险 | 控制措施 | 控制类型 |
|----------|----------|----------|----------|
| 费用管理 | 费用超支、虚假费用、审批不当 | 费用审批、真实性审核、预算控制 | 预防性+检查性 |
| 预算管理 | 预算不准、执行偏差、调整随意 | 预算审批、执行监控、调整控制 | 预防性+检查性 |
| 合同管理 | 合同风险、条款不当、执行不力 | 合同审核、法务审查、执行监督 | 预防性+检查性 |
| 决策管理 | 决策失误、程序不当、执行偏差 | 决策程序、集体决策、执行监控 | 预防性+检查性 |

### 容易出错的领域

1. **费用审核**：费用真实性审核不严格，虚假费用未能识别
2. **预算控制**：预算执行监控不及时，超预算支出控制不力
3. **合同管理**：合同条款审核不充分，合同风险识别不到位
4. **重大决策**：决策程序不规范，决策依据不充分
5. **档案管理**：管理档案不完整，重要文件保管不当

## ⚠️ 常见问题与解决方案

### 问题1：如何确定控制测试范围？
**解决方案**：根据风险评估结果和重要性水平确定，通常包括关键业务流程的控制点。

### 问题2：如何处理控制偏差？
**解决方案**：分析偏差原因，评估偏差影响，必要时调整审计策略。

### 问题3：如何选择样本量？
**解决方案**：根据控制频率、重要性水平和可接受偏差率确定样本量。

### 问题4：如何评价控制有效性？
**解决方案**：综合考虑控制设计、执行情况和偏差率等因素。

## C11-2 管理循环评价控制偏差

### 示例-评价控制偏差

**编制日期：2014.1.5**
**索引号：C11-2**  
**截止日：2020年12月31日**
**复核人：DDD**
**复核日期：2014.1.8**
**页次：**

#### 一、审计目标
汇总各流程中经测试发现的控制偏差，并得出结论。

#### 二、控制例外情况描述
经理层监督客户投诉的数量及类型并把这些信息作为估计销售退回的依据。然而，注册会计师在执行时发现某一期间计算出的投诉率没有包含某一地区收到的投诉。

#### 三、控制偏差评价步骤

| 评价步骤 | 是/否 | 评价分析过程 | 备注 |
|----------|-------|-------------|------|
| **步骤一**：确认发现的控制例外情况（选定的样本无法执行控制测试）是否属于控制偏差（内部控制运行偏离设计）？ | **是** | 该控制例外情况影响销售收入的金额，且金额并不关紧要。进入步骤二 | |
| **步骤二**：确定控制偏差的性质：系统性偏差？人为有意造成的偏差？随机性偏差？ | **随机性偏差** | 注册会计师确定这只是众多地区中唯一的一个没有报告投诉率的，而且其他所有月份都完成了投诉率的报告。确定应对措施：进入步骤三 | |
| **步骤三**：对于随机性的偏差，确定适当的审计应对措施是否进行测试？直接将该项控制偏差视作缺陷？ | **扩大样本量** | 注册会计师确定该项控制偏差是随机性的，并应扩大样本量来予以确认。进入步骤四 | |
| **步骤四**：根据发现的控制偏差的数量来确定扩大样本量的范围，扩大样本量进行测试还是否发现新的偏差？ | **否** | 尽管存在一项控制偏差，但测试范围内的错报率在可容忍范围内且该控制偏差不具有代表性。**控制有效** | |
| **步骤五**：评价控制缺陷。 | | | |

| 在评价控制偏差的性质时，需要考虑： |  |  |  |  |  |  |  |  |
| （1）控制偏差是否具有共同的特征。例如，控制偏差发生在某些特定类型的业务、地点或生产线中，或者发生在同样的人员 |  |  |  |  |  |  |  |  |
| 执行控制时，抑或是发生在某一特定的时期。 |  |  |  |  |  |  |  |  |
| （2）控制偏差是否是由于基础工作错误造成的。例如，控制偏差是由于错误的公司或计算造成的，或者是基于不完整或 |  |  |  |  |  |  |  |  |
| 不准确的数据造成的。 |  |  |  |  |  |  |  |  |
| 如果确认偏差时系统性的或人为有意造成的，应直接将该控制偏差视为一项内部控制缺陷。 |  |  |  |  |  |  |  |  |
| 注3：随机性偏差的应对措施 |  |  |  |  |  |  |  |  |
| 如果认为一项控制偏差是随机的，则应当确定适当的审计应对措施。包括： |  |  |  |  |  |  |  |  |
| （1）扩大样本量进行测试，从而确定测试发现的错误率在可容忍范围内； |  |  |  |  |  |  |  |  |
| （2）直接将该项控制偏差视作一项内部控制缺陷。 |  |  |  |  |  |  |  |  |
| 如果决定扩大样本量进行测试，注册会计师应当根据发现的控制偏差的数量来确定扩大样本量的范围。例如，对于每日发生 |  |  |  |  |  |  |  |  |
| 多次的控制，如果初始样本规模为25个，当测试发现一项控制偏差，且该偏差不是系统性偏差时，注册会计师扩大样本规模 |  |  |  |  |  |  |  |  |
| 进行测试，所增加的样本规模至少为15个。 |  |  |  |  |  |  |  |  |
| 注4：扩大样本量后的测试结果 |  |  |  |  |  |  |  |  |
| 如果扩大样本量测试后再次发现偏差，则注册会计师可以得出该控制无效的结论。 |  |  |  |  |  |  |  |  |
| 如果扩大样本量测试没有再次发现偏差，则注册会计师可以得出该控制有效的结论。即尽管存在一项控制偏差，但测试范围 |  |  |  |  |  |  |  |  |
| 内的错误率在可容忍范围内且该控制偏差不具有代表性。 |  |  |  |  |  |  |  |  |
| 单项控制并非一定要毫无偏差地运行，才被认为有效。 |  |  |  |  |  |  |  |  |
| 注5：评价是否构成设计缺陷 |  |  |  |  |  |  |  |  |
| 若确定发现的控制例外情况不属于控制偏差，则可以选取替代样本以获得证明控制运行有效的充分、适当的审计证据。 |  |  |  |  |  |  |  |  |
| 但需确定控制例外情况是否表明相关控制的设计存在缺陷。 |  |  |  |  |  |  |  |  |
| 设计缺陷是指缺少为实现控制目标所必需的控制，或现有控制设计不适当、即使正常运行也难以实现预期的控制目标。 |  |  |  |  |  |  |  |  |
