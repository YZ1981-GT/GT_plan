# C4 存货业务循环控制测试底稿模板库

## 📖 文档概述

本文档为存货业务循环控制测试底稿模板库，包含存货相关的所有控制测试程序模板，适用于致同会计师事务所的审计工作。

## 🚀 使用说明

1. **控制测试选择**：根据被审计单位的实际情况选择合适的控制测试模板
2. **程序执行**：按照底稿中的控制测试程序逐步执行
3. **记录完整**：确保所有控制测试程序都有完整的执行记录
4. **偏差处理**：及时识别和处理控制偏差

## 📋 标准化选项清单

### 🎯 认定维度选项
- **存在/发生**：确认存货交易和事项确实发生，存货确实存在
- **完整性**：确认所有应当记录的存货交易和事项均已记录
- **准确性、计价和分摊**：确认存货交易和事项以恰当的金额记录
- **权利和义务**：确认企业拥有或控制存货相关的权利
- **截止**：确认存货交易和事项记录于正确的会计期间
- **分类**：确认存货交易和事项记录于恰当的账户
- **列报**：确认存货在财务报表中得到恰当列报和披露

### ⚙️ 控制属性选项
- **人工的**：完全由人工执行的控制活动
- **自动化的**：由信息系统自动执行的控制活动
- **人工依赖信息系统控制**：依赖信息系统产生信息的人工控制

### ⏰ 频率选项
- **每笔交易**：针对每笔存货交易执行的控制
- **每天**：每个工作日执行一次的控制
- **每周**：每周执行一次的控制
- **每半月**：每半个月执行一次的控制
- **每月**：每月执行一次的控制
- **每季度**：每季度执行一次的控制
- **每年**：每年执行一次的控制
- **非常规/低运行频率**：不定期或根据特定情况执行的控制
- **其他（描述）**：其他特定频率的控制

### 🎚️ 与控制相关的风险等级
- **高**：控制失效可能导致重大错报风险
- **中**：控制失效可能导致中等程度错报风险
- **低**：控制失效导致的错报风险相对较低

### 🔍 控制测试方法详细说明

#### 1. 询问
**定义**：是指向被审计单位相关人员进行询问
**适用情况**：了解控制设计和执行情况
**执行要点**：
- 选择合适的询问对象
- 准备结构化的询问提纲
- 获取书面确认（如需要）

#### 2. 检查
**定义**：是指对被审计单位相关文件或记录进行检查
**适用情况**：验证控制执行的书面证据
**执行要点**：
- 检查原始凭证和记录
- 关注审批签字和日期
- 验证文件的完整性和真实性

#### 3. 观察
**定义**：是指察看相关人员执行某项控制活动的过程
**适用情况**：了解控制的实际执行过程
**执行要点**：
- 在正常业务环境下观察
- 记录观察到的关键控制步骤
- 注意控制执行的一致性

#### 4. 重新执行
**定义**：是指独立执行作为被审计单位内部控制组成部分的程序或控制
**适用情况**：验证控制的有效性
**执行要点**：
- 使用相同的数据和参数
- 比较执行结果的一致性
- 分析差异原因

#### 5. 前期
**定义**：利用以前审计获取的某些控制测试的审计证据
**适用情况**：控制未发生变化的情况
**执行要点**：
- 评估控制变化情况
- 确定可依赖的程度
- 补充必要的当期测试

#### 6. 前推
**定义**：利用期中已获取的控制测试证据推断至期末
**适用情况**：期中测试基础上的期末推断
**执行要点**：
- 评估剩余期间的控制变化
- 执行补充测试程序
- 考虑控制环境变化

#### 7. 利用内部审计工作
**定义**：在某些情况下，注册会计师可以利用内部审计人员的工作
**适用情况**：内部审计工作质量可靠的情况
**执行要点**：
- 评估内部审计的客观性和胜任能力
- 复核内部审计工作质量
- 确定利用程度

#### 8. 利用服务机构的审计师的报告
**定义**：服务注册会计师对服务机构控制的报告
**适用情况**：使用服务机构处理存货业务的情况
**执行要点**：
- 评估报告的相关性和可靠性
- 关注报告涵盖的期间
- 考虑补充控制的必要性

1. **采购控制**：存在/发生 + 权利和义务
2. **入库验收控制**：完整性 + 准确性、计价和分摊
3. **库存盘点控制**：存在/发生 + 完整性
4. **出库控制**：存在/发生 + 截止 + 分类

### 选择建议
- **高价值存货**：建议选择"高"风险等级，增加测试样本量
- **自动化程度高**：重点关注"人工依赖信息系统控制"
- **复杂存货计价**：重点测试"准确性、计价和分摊"认定

## 📊 存货循环业务流程图及描述

### 存货循环主要业务流程

```
采购需求 → 采购申请 → 采购审批 → 采购执行 → 入库验收 → 入库登记 → 库存管理 → 出库申请 → 出库审批 → 出库执行 → 库存盘点
    ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓
  生产部     采购部     采购部     采购部     仓库部     仓库部     仓库部     销售部     仓库部     仓库部     仓库部
```

### 详细流程描述

**A. 采购管理阶段**
1. **采购需求**：各部门根据生产计划或库存情况提出采购需求
2. **采购申请**：采购部门编制采购申请，明确采购物品、数量、规格等
3. **采购审批**：按照审批权限对采购申请进行审批
4. **供应商选择**：选择合格供应商，进行询价比价
5. **采购执行**：签订采购合同，执行采购业务

**B. 入库管理阶段**
1. **到货验收**：仓库部门对到货物品进行验收
2. **质量检验**：质检部门对物品质量进行检验
3. **入库登记**：合格物品办理入库手续，登记入库记录
4. **单据传递**：入库单据及时传递给财务部门
5. **账务处理**：财务部门及时进行账务处理

**C. 库存管理阶段**
1. **库存分类**：按照物品性质进行分类存放
2. **库存标识**：对库存物品进行标识和编号
3. **库存维护**：定期维护库存物品，防止损坏
4. **库存监控**：实时监控库存数量和状态
5. **安全管理**：确保库存物品的安全

**D. 出库管理阶段**
1. **出库申请**：各部门根据需要提出出库申请
2. **出库审批**：按照权限对出库申请进行审批
3. **出库执行**：仓库部门按照审批的出库单执行出库
4. **出库登记**：及时登记出库记录
5. **账务处理**：财务部门及时进行出库账务处理

**E. 盘点管理阶段**
1. **盘点计划**：制定库存盘点计划
2. **盘点执行**：按计划执行库存盘点
3. **盘点记录**：详细记录盘点结果
4. **差异分析**：分析盘点差异原因
5. **差异处理**：及时处理盘点差异

### 关键控制点识别

| 业务环节 | 关键风险 | 控制措施 | 控制类型 |
|----------|----------|----------|----------|
| 采购管理 | 采购舞弊、价格不合理、质量问题 | 采购审批、供应商管理、比价采购 | 预防性 |
| 入库管理 | 验收不严、单据不符、账实不符 | 验收制度、三单匹配、及时入账 | 预防性+检查性 |
| 库存管理 | 存货损失、盗窃、损坏 | 分类存放、安全管理、定期检查 | 预防性+检查性 |
| 出库管理 | 未经授权出库、出库错误 | 出库审批、单据控制、及时记录 | 预防性 |
| 盘点管理 | 盘点不准、差异处理不当 | 定期盘点、差异分析、及时调整 | 检查性+纠正性 |

### 容易出错的领域

1. **入库验收**：验收程序不严格，质量把关不严
2. **库存盘点**：盘点不及时或不准确，账实不符
3. **出库控制**：出库审批不规范，存在未经授权出库
4. **存货计价**：计价方法不一致，成本核算不准确
5. **存货减值**：减值测试不充分，减值准备计提不足

#### 四、审计说明

**控制测试执行要点：**

1. **采购申请控制测试**
   - 检查采购申请是否经过适当审批
   - 验证采购需求的合理性和必要性
   - 确认采购申请程序的完整性

2. **入库验收控制测试**
   - 观察入库验收流程的执行
   - 检查验收记录的完整性和准确性
   - 验证三单匹配（采购单、入库单、发票）

3. **库存盘点控制测试**
   - 观察库存盘点流程的实际执行
   - 检查盘点记录的准确性
   - 验证盘点差异的处理程序

4. **出库控制测试**
   - 检查出库审批程序的执行
   - 验证出库单据的完整性
   - 确认出库记录的及时性

5. **存货计价控制测试**
   - 检查存货计价方法的一致性
   - 验证成本核算的准确性
   - 确认存货减值测试的充分性

**重点关注领域：**
- 入库验收程序的严格执行
- 库存盘点制度的有效性
- 出库审批权限的合理设置
- 存货安全保管措施
- 存货计价和减值测试

**常见控制缺陷：**
- 入库验收程序不严格
- 库存盘点不及时或不准确
- 出库审批权限不清晰
- 存货保管措施不完善
- 存货减值测试不充分

## 🔄 控制偏差评价步骤

### 步骤一：
确认发现的控制例外情况（选定的样本无法执行控制测试）是否属于控制偏差（内控制运行偏离设计）？

| 是/否 | 评价分析过程 | 备注 |
|-------|-------------|------|
| | 该控制例外情况影响销售收入的金额，且金额与其他关紧要。进入步骤二 | |

### 步骤二：
确定控制偏差的性质：系统性偏差？人为有意造成的偏差？随机性偏差？

| 随机性偏差 | 注册会计师确定这只是众多地区中唯一一个没有报告投诉案的，而且其他所有月份都完成了投诉率的报告。确定应对措施：进入步骤三 |
|------------|----------------------------------------------------------------------------|

### 步骤三：
对于随机性的偏差，确定适当的审计应对措施是否需要进行测试？直接将该项控制偏差视作缺陷？

| 扩大样本量 | 注册会计师确定该项控制偏差是随机性的，并应扩大样本量来予以确认。进入步骤四 |
|------------|----------------------------------------------------------------------------|

### 步骤四：
根据发现的控制偏差的数量来确定扩大样本量的范围，扩大样本量进行测试是否发现新的偏差？

| 否 | 尽管存在一项控制偏差，但测试范围内的错误率在可容忍范围内且该控制偏差不具有代表性。控制有效 |
|----|---------------------------------------------------------------------------------------------|

### 步骤五：
评价控制缺陷。

### 步骤六：
若确定发现的控制例外情况不属于控制偏差，则可选取替代样本以获得证明控制运行有效的充分、适当的审计证据。
但需确定控制例外情况是否表明相关控制的设计存在缺陷？

#### 三、控制例外情况描述
经理层监督客户投诉的数量及类型并把这些信息作为估计销售退回的依据。然而，注册会计师在执行时发现某一期间计算出的投诉率没有包含某一地区收到的投诉。

#### 四、评价结论
该控制例外情况属于随机性偏差，扩大样本测试后未发现偏差，控制有效

| 对于一个选定的样本或替代性样本，如果无法执行注册会计师所设计的控制测试或适当的替代性测试，则注册会计师应认为 |  |  |  |  |  |  |  | 该控制例外情况属于系统性偏差，构成控制缺陷 |
| 该样本是一项控制例外情况。 |  |  |  |  |  |  |  |  |
| 例如，对于一个选定的样本，如果由于企业不能提供支持性文档而无法执行控制测试，则其属于一项控制例外情况并需要 |  |  |  |  |  |  |  |  |
| 确定适当的应对措施。 |  |  |  |  |  |  |  |  |
| 对于发现的控制例外情况，首先应当确定其是否构成一项控制偏差。 |  |  |  |  |  |  |  |  |
| 当出现以下情况时，一般可以认为控制例外情况不是控制偏差： |  |  |  |  |  |  |  |  |
| （1）根据进一步的了解和评价，注册会计师确定识别出的例外情况事实上不是对控制预设目标的偏离，即注册会计师原先 |  |  |  |  |  |  |  |  |
| 对重大业务流程及相关控制的理解有误。 |  |  |  |  |  |  |  |  |
| （2）基于对控制设计的了解，注册会计师能够通过追加执行除询问以外的审计程序获取充分、适当的审计证据，以确认该 |  |  |  |  |  |  |  |  |
| 例外情况引发的错误在可以接受的水平，即小于临界值（明显微小的错报金额SAD）。 |  |  |  |  |  |  |  |  |
| 控制偏差 |  |  |  |  |  |  |  |  |
| 控制偏差即内部控制运行偏离设计的情况，是指在内部控制测试中，注册会计师选择进行测试的某个控制没有按照设计的 |  |  |  |  |  |  |  |  |
| 要求被有效执行或者没有被执行。 |  |  |  |  |  |  |  |  |
| 发生控制偏差往往是由于执行控制的人员在收集或处理数据时发生错误、没有进行适当的跟进，或者没有足够的能力和意愿 |  |  |  |  |  |  |  |  |
| 去执行有效的控制。 |  |  |  |  |  |  |  |  |
| 注2：控制偏差的性质 |  |  |  |  |  |  |  |  |
| 系统性偏差是指在相似情形下，预计都会发生的错误。 |  |  |  |  |  |  |  |  |
| 随机性偏差是指偶然发生的错误。随机性偏差是由某一孤立事件引起的，该事件只有在特定条件下才会重复发生，因而 |  |  |  |  |  |  |  |  |
| 随机性偏差对总体误差不具有代表性。 |  |  |  |  |  |  |  |  |
| 如果发现的控制偏差是系统性偏差或人为有意造成的偏差，注册会计师应当考虑是否存在舞弊的迹象以及对审计方案的影响。 |  |  |  |  |  |  |  |  |
| 注册会计师在将控制偏差确认为随机性的时候需要谨慎，在得出该结论之前，应当实施追加的审计程序，以高度确信该偏差 |  |  |  |  |  |  |  |  |
| 对总体误差不具有代表性。 |  |  |  |  |  |  |  |  |
| 在评价控制偏差的性质时，需要考虑： |  |  |  |  |  |  |  |  |
| （1）控制偏差是否具有共同的特征。例如，控制偏差发生在某些特定类型的业务、地点或生产线中，或者发生在同样的人员 |  |  |  |  |  |  |  |  |
| 执行控制时，抑或是发生在某一特定的时期。 |  |  |  |  |  |  |  |  |
| （2）控制偏差是否是由于基础工作错误造成的。例如，控制偏差是由于错误的公司或计算造成的，或者是基于不完整或 |  |  |  |  |  |  |  |  |
| 不准确的数据造成的。 |  |  |  |  |  |  |  |  |
| 如果确认偏差时系统性的或人为有意造成的，应直接将该控制偏差视为一项内部控制缺陷。 |  |  |  |  |  |  |  |  |
| 注3：随机性偏差的应对措施 |  |  |  |  |  |  |  |  |
| 如果认为一项控制偏差是随机的，则应当确定适当的审计应对措施。包括： |  |  |  |  |  |  |  |  |
| （1）扩大样本量进行测试，从而确定测试发现的错误率在可容忍范围内； |  |  |  |  |  |  |  |  |
| （2）直接将该项控制偏差视作一项内部控制缺陷。 |  |  |  |  |  |  |  |  |
| 如果决定扩大样本量进行测试，注册会计师应当根据发现的控制偏差的数量来确定扩大样本量的范围。例如，对于每日发生 |  |  |  |  |  |  |  |  |
| 多次的控制，如果初始样本规模为25个，当测试发现一项控制偏差，且该偏差不是系统性偏差时，注册会计师扩大样本规模 |  |  |  |  |  |  |  |  |
| 进行测试，所增加的样本规模至少为15个。 |  |  |  |  |  |  |  |  |
| 注4：扩大样本量后的测试结果 |  |  |  |  |  |  |  |  |
| 如果扩大样本量测试后再次发现偏差，则注册会计师可以得出该控制无效的结论。 |  |  |  |  |  |  |  |  |
| 如果扩大样本量测试没有再次发现偏差，则注册会计师可以得出该控制有效的结论。即尽管存在一项控制偏差，但测试范围 |  |  |  |  |  |  |  |  |
| 内的错误率在可容忍范围内且该控制偏差不具有代表性。 |  |  |  |  |  |  |  |  |
| 单项控制并非一定要毫无偏差地运行，才被认为有效。 |  |  |  |  |  |  |  |  |
| 注5：评价是否构成设计缺陷 |  |  |  |  |  |  |  |  |
| 若确定发现的控制例外情况不属于控制偏差，则可以选取替代样本以获得证明控制运行有效的充分、适当的审计证据。 |  |  |  |  |  |  |  |  |
| 但需确定控制例外情况是否表明相关控制的设计存在缺陷。 |  |  |  |  |  |  |  |  |
| 设计缺陷是指缺少为实现控制目标所必需的控制，或现有控制设计不适当、即使正常运行也难以实现预期的控制目标。 |  |  |  |  |  |  |  |  |
