# C8 无形资产及其他长期资产循环控制测试底稿模板库

## 📖 文档概述

本文档为无形资产及其他长期资产循环控制测试底稿模板库，包含无形资产及其他长期资产相关的所有控制测试程序模板，适用于致同会计师事务所的审计工作。

## 🚀 使用说明

1. **控制测试选择**：根据被审计单位的实际情况选择合适的控制测试模板
2. **程序执行**：按照底稿中的控制测试程序逐步执行
3. **记录完整**：确保所有控制测试程序都有完整的执行记录
4. **偏差处理**：及时识别和处理控制偏差

## 📋 标准化选项清单

### 🎯 认定维度选项
- **存在/发生**：确认记录的无形资产及其他长期资产交易和事项已发生且与被审计单位有关
- **完整性**：确认所有应当记录的无形资产及其他长期资产交易和事项均已记录
- **准确性、计价和分摊**：确认与无形资产及其他长期资产有关的金额及其他数据已恰当记录，相关资产以恰当的金额计入财务报表
- **权利和义务**：确认被审计单位拥有或控制无形资产及其他长期资产的权利
- **截止**：确认无形资产及其他长期资产交易和事项记录于正确的会计期间
- **分类**：确认无形资产及其他长期资产在财务报表中被恰当地分类
- **列报**：确认无形资产及其他长期资产在财务报表中被恰当地列报和披露

### ⚙️ 控制属性选项
- **人工的**：完全由人工执行的控制活动，需要人工判断和操作
- **自动化的**：由信息系统自动执行的控制活动，无需人工干预
- **人工依赖信息系统控制**：人工控制活动依赖于信息系统生成的数据或报告

### ⏰ 频率选项
- **每笔交易**：针对每笔无形资产及其他长期资产交易执行的控制
- **每天**：每个工作日执行一次的控制
- **每周**：每周执行一次的控制
- **每半月**：每两周执行一次的控制
- **每月**：每月执行一次的控制
- **每季度**：每季度执行一次的控制
- **每年**：每年执行一次的控制
- **非常规/低运行频率**：不定期或按特殊需求执行的控制
- **其他（描述）**：需要具体说明的其他频率

### 🎚️ 与控制相关的风险等级
- **高**：该控制直接应对高风险领域，失效可能导致财务报表重大错报
- **中**：该控制应对中等风险，失效可能导致一定程度的错报
- **低**：该控制应对低风险领域，失效对财务报表影响较小

### 🔍 控制测试方法详细说明

#### 1. 询问
**定义**：是指向被审计单位相关人员进行询问
**适用情况**：了解控制设计和执行情况
**执行要点**：
- 选择合适的询问对象
- 准备结构化的询问提纲
- 获取书面确认（如需要）

#### 2. 检查
**定义**：是指对被审计单位相关文件或记录进行检查
**适用情况**：验证控制执行的书面证据
**执行要点**：
- 检查原始凭证和记录
- 关注审批签字和日期
- 验证文件的完整性和真实性

#### 3. 观察
**定义**：是指察看相关人员执行某项控制活动的过程
**适用情况**：了解控制的实际执行过程
**执行要点**：
- 在正常业务环境下观察
- 记录观察到的关键控制步骤
- 注意控制执行的一致性

#### 4. 重新执行
**定义**：是指独立执行作为被审计单位内部控制组成部分的程序或控制
**适用情况**：验证控制的有效性
**执行要点**：
- 使用相同的数据和参数
- 比较执行结果的一致性
- 分析差异原因

#### 5. 前期
**定义**：利用以前审计获取的某些控制测试的审计证据
**适用情况**：控制未发生变化的情况
**执行要点**：
- 评估控制变化情况
- 确定可依赖的程度
- 补充必要的当期测试

#### 6. 前推
**定义**：利用期中已获取的控制测试证据推断至期末
**适用情况**：期中测试基础上的期末推断
**执行要点**：
- 评估剩余期间的控制变化
- 执行补充测试程序
- 考虑控制环境变化

#### 7. 利用内部审计工作
**定义**：在某些情况下，注册会计师可以利用内部审计人员的工作
**适用情况**：内部审计工作质量可靠的情况
**执行要点**：
- 评估内部审计的客观性和胜任能力
- 复核内部审计工作质量
- 确定利用程度

#### 8. 利用服务机构的审计师的报告
**定义**：服务注册会计师对服务机构控制的报告
**适用情况**：使用服务机构处理无形资产业务的情况
**执行要点**：
- 评估报告的相关性和可靠性
- 关注报告涵盖的期间
- 考虑补充控制的必要性

## 📚 标准化选项使用指南

### 无形资产及其他长期资产循环常用组合
1. **资产取得控制**：存在/发生 + 权利和义务
2. **摊销计提控制**：准确性、计价和分摊 + 截止
3. **减值测试控制**：准确性、计价和分摊 + 列报
4. **资产处置控制**：存在/发生 + 截止 + 分类

### 选择建议
- **重要无形资产**：建议选择"高"风险等级，重点关注权利和义务认定
- **自研无形资产**：重点测试"准确性、计价和分摊"认定和研发支出资本化控制
- **外购无形资产**：重点关注"存在/发生"和"权利和义务"认定

## 📊 无形资产循环业务流程图及描述

### 无形资产循环主要业务流程

```
资产需求 → 取得申请 → 取得审批 → 资产确认 → 初始计量 → 后续计量 → 摊销计提 → 减值测试 → 资产处置 → 档案管理 → 合规管理
    ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓
  研发部     法务部     管理层     财务部     财务部     财务部     财务部     财务部     法务部     档案部     合规部
```

### 详细流程描述

**A. 无形资产取得阶段**
1. **需求识别**：识别无形资产取得需求和目标
2. **取得申请**：编制无形资产取得申请
3. **取得审批**：按照审批权限进行取得审批
4. **资产确认**：确认无形资产的确认条件
5. **初始计量**：进行无形资产初始计量

**B. 无形资产管理阶段**
1. **资产登记**：建立无形资产台账和档案
2. **后续计量**：进行无形资产后续计量
3. **摊销计提**：按照摊销政策计提摊销
4. **减值测试**：定期进行减值测试
5. **价值监控**：监控无形资产价值变化

**C. 研发管理阶段**
1. **研发立项**：研发项目立项和审批
2. **研发过程**：研发过程管理和监控
3. **成果确认**：研发成果的确认和评估
4. **费用归集**：研发费用的归集和核算
5. **成果保护**：研发成果的知识产权保护

**D. 处置管理阶段**
1. **处置申请**：提出无形资产处置申请
2. **处置审批**：按照权限进行处置审批
3. **处置执行**：执行无形资产处置
4. **处置确认**：确认处置收益或损失
5. **档案更新**：更新无形资产档案

### 关键控制点识别

| 业务环节 | 关键风险 | 控制措施 | 控制类型 |
|----------|----------|----------|----------|
| 资产取得 | 确认错误、计量不准、程序不当 | 确认审核、计量复核、程序控制 | 预防性 |
| 资产管理 | 摊销错误、减值不足、档案缺失 | 摊销审核、减值测试、档案管理 | 预防性+检查性 |
| 研发管理 | 费用归集错误、成果确认不当 | 费用审核、成果评估、过程监控 | 预防性+检查性 |
| 资产处置 | 处置损失、程序不当、未经授权 | 处置审批、评估制度、程序控制 | 预防性 |

### 容易出错的领域

1. **资产确认**：无形资产确认标准把握不准确
2. **摊销计提**：摊销方法选择不当，摊销期限确定不准确
3. **减值测试**：减值测试程序不充分，减值准备计提不足
4. **研发费用**：研发费用归集不准确，资本化条件判断错误
5. **知识产权**：知识产权保护不充分，档案管理不完善

## ⚠️ 常见问题与解决方案

### 问题1：如何确定控制测试范围？
**解决方案**：根据风险评估结果和重要性水平确定，通常包括关键业务流程的控制点。

### 问题2：如何处理控制偏差？
**解决方案**：分析偏差原因，评估偏差影响，必要时调整审计策略。

### 问题3：如何选择样本量？
**解决方案**：根据控制频率、重要性水平和可接受偏差率确定样本量。

### 问题4：如何评价控制有效性？
**解决方案**：综合考虑控制设计、执行情况和偏差率等因素。

经理层监督客户投诉的数量及类型并把这些信息作为估计销售退回的依据。然而，注册会计师在执行时发现某一期间计算出的投诉率没有包含某一地区收到的投诉。

#### 三、控制偏差评价步骤

| 评价步骤 | 是/否 | 评价分析过程 | 备注 |
|----------|-------|-------------|------|
| **步骤一**：确认发现的控制例外情况（选定的样本无法执行控制测试）是否属于控制偏差（内部控制运行偏离设计）？ | **是** | 该控制例外情况影响销售收入的金额，且金额并不关紧要。进入步骤二 | |
| **步骤二**：确定控制偏差的性质：系统性偏差？人为有意造成的偏差？随机性偏差？ | **随机性偏差** | 注册会计师确定这只是众多地区中唯一的一个没有报告投诉率的，而且其他所有月份都完成了投诉率的报告。确定应对措施：进入步骤三 | |
| **步骤三**：对于随机性的偏差，确定适当的审计应对措施是否进行测试？直接将该项控制偏差视作缺陷？ | **扩大样本量** | 注册会计师确定该项控制偏差是随机性的，并应扩大样本量来予以确认。进入步骤四 | |
| **步骤四**：根据发现的控制偏差的数量来确定扩大样本量的范围，扩大样本量进行测试还是否发现新的偏差？ | **否** | 尽管存在一项控制偏差，但测试范围内的错报率在可容忍范围内且该控制偏差不具有代表性。**控制有效** | |
| **步骤五**：评价控制缺陷。 | | | |
| **步骤六**：若确定发现的控制例外情况不属于控制偏差，则可选取替代样本以获得证明控制运行有效的充分、适当的审计证据。但需确定控制例外情况是否表明相关控制的设计存在缺陷？ | | | |

#### 四、评价结论
**该控制例外情况属于随机性偏差，扩大样本测试后未发现偏差，控制有效**

| 控制偏差 |  |  |  |  |  |  |  |  |
| 控制偏差即内部控制运行偏离设计的情况，是指在内部控制测试中，注册会计师选择进行测试的某个控制没有按照设计的 |  |  |  |  |  |  |  |  |
| 要求被有效执行或者没有被执行。 |  |  |  |  |  |  |  |  |
| 发生控制偏差往往是由于执行控制的人员在收集或处理数据时发生错误、没有进行适当的跟进，或者没有足够的能力和意愿 |  |  |  |  |  |  |  |  |
| 去执行有效的控制。 |  |  |  |  |  |  |  |  |
| 注2：控制偏差的性质 |  |  |  |  |  |  |  |  |
| 系统性偏差是指在相似情形下，预计都会发生的错误。 |  |  |  |  |  |  |  |  |
| 随机性偏差是指偶然发生的错误。随机性偏差是由某一孤立事件引起的，该事件只有在特定条件下才会重复发生，因而 |  |  |  |  |  |  |  |  |
| 随机性偏差对总体误差不具有代表性。 |  |  |  |  |  |  |  |  |
| 如果发现的控制偏差是系统性偏差或人为有意造成的偏差，注册会计师应当考虑是否存在舞弊的迹象以及对审计方案的影响。 |  |  |  |  |  |  |  |  |
| 注册会计师在将控制偏差确认为随机性的时候需要谨慎，在得出该结论之前，应当实施追加的审计程序，以高度确信该偏差 |  |  |  |  |  |  |  |  |
| 对总体误差不具有代表性。 |  |  |  |  |  |  |  |  |
| 在评价控制偏差的性质时，需要考虑： |  |  |  |  |  |  |  |  |
| （1）控制偏差是否具有共同的特征。例如，控制偏差发生在某些特定类型的业务、地点或生产线中，或者发生在同样的人员 |  |  |  |  |  |  |  |  |
| 执行控制时，抑或是发生在某一特定的时期。 |  |  |  |  |  |  |  |  |
| （2）控制偏差是否是由于基础工作错误造成的。例如，控制偏差是由于错误的公司或计算造成的，或者是基于不完整或 |  |  |  |  |  |  |  |  |
| 不准确的数据造成的。 |  |  |  |  |  |  |  |  |
| 如果确认偏差时系统性的或人为有意造成的，应直接将该控制偏差视为一项内部控制缺陷。 |  |  |  |  |  |  |  |  |
| 注3：随机性偏差的应对措施 |  |  |  |  |  |  |  |  |
| 如果认为一项控制偏差是随机的，则应当确定适当的审计应对措施。包括： |  |  |  |  |  |  |  |  |
| （1）扩大样本量进行测试，从而确定测试发现的错误率在可容忍范围内； |  |  |  |  |  |  |  |  |
| （2）直接将该项控制偏差视作一项内部控制缺陷。 |  |  |  |  |  |  |  |  |
| 如果决定扩大样本量进行测试，注册会计师应当根据发现的控制偏差的数量来确定扩大样本量的范围。例如，对于每日发生 |  |  |  |  |  |  |  |  |
| 多次的控制，如果初始样本规模为25个，当测试发现一项控制偏差，且该偏差不是系统性偏差时，注册会计师扩大样本规模 |  |  |  |  |  |  |  |  |
| 进行测试，所增加的样本规模至少为15个。 |  |  |  |  |  |  |  |  |
| 注4：扩大样本量后的测试结果 |  |  |  |  |  |  |  |  |
| 如果扩大样本量测试后再次发现偏差，则注册会计师可以得出该控制无效的结论。 |  |  |  |  |  |  |  |  |
| 如果扩大样本量测试没有再次发现偏差，则注册会计师可以得出该控制有效的结论。即尽管存在一项控制偏差，但测试范围 |  |  |  |  |  |  |  |  |
| 内的错误率在可容忍范围内且该控制偏差不具有代表性。 |  |  |  |  |  |  |  |  |
| 单项控制并非一定要毫无偏差地运行，才被认为有效。 |  |  |  |  |  |  |  |  |
| 注5：评价是否构成设计缺陷 |  |  |  |  |  |  |  |  |
| 若确定发现的控制例外情况不属于控制偏差，则可以选取替代样本以获得证明控制运行有效的充分、适当的审计证据。 |  |  |  |  |  |  |  |  |
| 但需确定控制例外情况是否表明相关控制的设计存在缺陷。 |  |  |  |  |  |  |  |  |
| 设计缺陷是指缺少为实现控制目标所必需的控制，或现有控制设计不适当、即使正常运行也难以实现预期的控制目标。 |  |  |  |  |  |  |  |  |
