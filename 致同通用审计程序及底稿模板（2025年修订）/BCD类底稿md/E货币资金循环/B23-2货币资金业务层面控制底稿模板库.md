# B23-2 货币资金业务层面控制底稿模板库

## 文档概述

本文档为致同会计师事务所货币资金业务层面控制底稿模板库，包含了解、评价并测试货币资金业务层面控制的所有相关底稿模板，适用于致同会计师事务所的审计工作。

## 使用说明

1. **控制了解**：根据被审计单位的实际情况了解货币资金内部控制
2. **控制评价**：按照底稿中的控制评价程序逐步执行
3. **控制测试**：确保所有控制测试程序都有完整的执行记录
4. **结果汇总**：及时汇总控制了解和测试结果

## 📚 底稿间关系说明

### 底稿执行顺序
1. **B23-2** → 总体程序表，指导整个审计过程

### 控制类型参考示例（不归档）

#### 一、控制类型分类

| 控制类型 | 定义 | 例子 |
|----------|------|------|
| **预防性控制** | 旨在防止错误或舞弊发生 | • 职责分离：出纳员不能同时负责现金收付和记账<br>• 授权审批：所有现金支出必须经过财务经理审批<br>• 物理控制：现金存放在保险柜中<br>• 系统控制：系统自动检查银行账户余额 |
| **检查性控制** | 旨在发现已经发生但未被预防的错误或舞弊 | • 对账：银行存款余额调节表<br>• 复核：财务经理复核现金日记账<br>• 分析性复核：分析现金余额与去年同期比较<br>• 盘点：定期对现金进行盘点 |
| **人工控制** | 由人工执行的控制活动 | • 财务经理审批现金支出<br>• 出纳员进行现金盘点<br>• 会计员编制银行调节表 |
| **自动控制** | 由信息系统自动执行的控制活动 | • 系统自动检查银行账户余额<br>• 系统自动生成银行对账单<br>• 系统自动计算现金余额 |
| **混合控制** | 结合了人工和自动元素的控制活动 | • 系统生成异常报告，人工进行复核<br>• 系统自动计算，人工进行审批<br>• 系统自动对账，人工调查差异 |
| **一般控制** | 适用于所有信息系统的控制 | • 系统访问权限管理<br>• 数据备份和恢复<br>• 系统开发变更控制 |
| **应用控制** | 针对特定应用系统的控制 | • 现金收付系统的数据校验<br>• 银行系统的序列号控制<br>• 票据系统的完整性检查 |

#### 二、控制频率分类

| 控制频率 | 定义 | 例子 |
|----------|------|------|
| **每笔** | 每发生一笔交易时执行 | • 现金支出审批<br>• 银行转账审批<br>• 票据开具审批 |
| **每日** | 每天执行一次 | • 现金盘点<br>• 银行余额核对<br>• 资金监控报告 |
| **每周** | 每周执行一次 | • 现金周报<br>• 银行账户检查<br>• 资金流向分析 |
| **每月** | 每月执行一次 | • 银行对账<br>• 现金月报<br>• 资金使用分析 |
| **每季度** | 每季度执行一次 | • 银行账户权限复核<br>• 资金管理制度检查<br>• 内部控制评价 |
| **每年** | 每年执行一次 | • 银行账户年检<br>• 资金管理制度更新<br>• 内部控制审计 |
| **不定期** | 根据需要进行 | • 异常情况调查<br>• 紧急资金调拨<br>• 特殊业务处理 |
