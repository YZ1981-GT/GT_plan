# C3 货币资金业务循环控制测试底稿模板库

## 文档概述

本文档为货币资金业务循环控制测试底稿模板库，包含货币资金相关的所有控制测试程序模板，适用于致同会计师事务所的审计工作。

## 使用说明

1. **控制测试选择**：根据被审计单位的实际情况选择合适的控制测试模板
2. **程序执行**：按照底稿中的控制测试程序逐步执行
3. **记录完整**：确保所有控制测试程序都有完整的执行记录
4. **偏差处理**：及时识别和处理控制偏差

## 📊 货币资金循环业务流程图及描述

### 货币资金循环主要业务流程

```
资金需求 → 资金申请 → 资金审批 → 资金支付 → 银行对账 → 现金盘点 → 资金监控 → 资金调拨 → 投资管理 → 风险控制
    ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓
  各部门     申请部门    财务部     财务部     财务部     出纳      财务部     财务部     投资部     风控部
```

### 详细流程描述

**A. 现金管理阶段**
1. **现金收付**：严格按照现金管理制度执行现金收付业务
2. **现金盘点**：定期进行现金盘点，确保账实相符
3. **现金保管**：现金存放在保险柜中，确保安全
4. **现金限额**：严格控制现金库存限额，超限及时存银行
5. **现金记录**：及时准确记录现金收付业务

**B. 银行存款管理阶段**
1. **银行账户管理**：规范银行账户开立、变更、撤销程序
2. **银行对账**：定期进行银行对账，及时发现差异
3. **银行印鉴管理**：银行印鉴分人保管，确保安全
4. **银行余额监控**：实时监控银行账户余额变化
5. **银行手续费管理**：合理控制银行手续费支出

**C. 资金支付阶段**
1. **支付申请**：各部门提出资金支付申请
2. **支付审批**：按照审批权限进行资金支付审批
3. **支付执行**：财务部门执行资金支付业务
4. **支付记录**：及时准确记录资金支付情况
5. **支付监控**：监控资金支付的合规性和安全性

**D. 资金调拨阶段**
1. **调拨申请**：提出资金调拨申请
2. **调拨审批**：按照权限审批资金调拨
3. **调拨执行**：执行资金调拨业务
4. **调拨记录**：记录资金调拨情况
5. **调拨监控**：监控资金调拨的效果

### 关键控制点识别

| 业务环节 | 关键风险 | 控制措施 | 控制类型 |
|----------|----------|----------|----------|
| 现金管理 | 现金丢失、挪用、盘点差异 | 现金盘点、保管控制、限额控制 | 预防性+检查性 |
| 银行管理 | 银行账户风险、对账差异 | 印鉴分管、定期对账、账户管理 | 预防性+检查性 |
| 资金支付 | 未经授权支付、支付错误 | 支付审批、权限控制、复核制度 | 预防性 |
| 资金调拨 | 调拨风险、资金安全 | 调拨审批、监控制度、记录管理 | 预防性+检查性 |

### 容易出错的领域

1. **现金管理**：现金盘点不及时，账实不符
2. **银行对账**：银行对账不及时，未及时发现差异
3. **支付审批**：支付审批权限不清晰，存在越权审批
4. **印鉴管理**：银行印鉴保管不当，存在安全风险
5. **资金监控**：资金流向监控不充分，存在资金风险

## 📋 标准化选项清单

### 认定维度选项
| 认定类型 | 说明 |
|----------|------|
| 存在/发生 | 记录的交易和事项已经发生，且与被审计单位有关 |
| 完整性 | 所有应当记录的交易和事项均已记录 |
| 准确性、计价和分摊 | 交易和事项以恰当的金额包括在财务报表中，收入或费用被分摊至恰当的期间 |
| 权利和义务 | 被审计单位拥有或控制记录的资产的权利，负债是被审计单位的义务 |
| 截止 | 交易和事项记录于正确的会计期间 |
| 分类 | 交易和事项记录于恰当的账户 |
| 列报 | 交易和事项在财务报表中被恰当列报和披露 |

### 控制属性选项
| 控制属性 | 说明 |
|----------|------|
| 人工的 | 由人工执行的控制活动 |
| 自动化的 | 由信息系统自动执行的控制活动 |
| 人工依赖信息系统控制 | 人工控制依赖于信息系统产生的信息 |

### 频率选项
| 频率类型 | 说明 |
|----------|------|
| 每笔交易 | 对每笔交易都执行的控制 |
| 每天 | 每日执行的控制 |
| 每周 | 每周执行的控制 |
| 每半月 | 每半月执行的控制 |
| 每月 | 每月执行的控制 |
| 每季度 | 每季度执行的控制 |
| 每年 | 每年执行的控制 |
| 非常规/低运行频率 | 不定期或低频率执行的控制 |
| 其他（描述） | 其他特殊频率的控制 |

### 与控制相关的风险等级
| 风险等级 | 说明 |
|----------|------|
| 高 | 控制失效对财务报表产生重大影响的可能性高 |
| 中 | 控制失效对财务报表产生中等程度影响 |
| 低 | 控制失效对财务报表影响相对较小 |

### 控制测试方法详细说明
| 测试方法 | 详细说明 |
|----------|----------|
| 询问 | 是指向被审计单位的内部或外部的知情人员获取财务信息和非财务信息 |
| 检查 | 是指对被审计单位的内部或外部生成的，以纸质、电子或其他介质形式存在的记录和文件进行审查，或对资产进行实物检查 |
| 观察 | 是指察看相关人员正在从事的活动或实施的程序 |
| 重新执行 | 是指独立执行作为被审计单位内部控制组成部分的控制或检测程序 |
| 前期 | 利用以前审计期间获取的审计证据，如果前期审计期间获取的有关控制运行有效性的审计证据 |
| 前推 | 利用期中获取的审计证据，通过实施期中获取或更新审计证据，可能会在期末之前执行控制测试 |
| 利用内部审计工作 | 在某些情况下，可以利用内部审计师的工作来获取控制测试 |
| 利用服务机构的审计师 | 服务注册会计师向第三方报告可能为与服务组织控制的运行有效性提供相关审计证据 |

## 📋 标准化选项使用指南

### 如何使用认定维度选项
1. **选择相关认定**：根据控制点的性质选择1-3个最相关的认定
2. **常见组合**：
   - 现金收付控制：存在/发生 + 完整性
   - 银行对账控制：存在/发生 + 完整性 + 准确性
   - 资金审批控制：存在/发生 + 准确性

### 如何确定控制属性
- **人工的**：需要人工判断和执行的控制（如审批、复核）
- **自动化的**：由系统自动执行的控制（如系统权限控制）
- **人工依赖信息系统控制**：人工控制但依赖系统信息（如资金监控）

### 如何选择测试方法
- **检查**：适用于有书面证据的控制（如审批单据）
- **观察**：适用于现场执行的控制（如现金盘点）
- **询问**：适用于了解控制执行情况
- **重新执行**：适用于验证控制有效性

### 如何确定样本量
参考**控制测试抽样指引表**：
- 根据控制频率确定基础样本量
- 考虑控制重要性和风险等级调整
- 高风险控制适当增加样本量

## ⚠️ 常见问题与解决方案

### 问题1：如何确定控制测试范围？
**解决方案**：根据风险评估结果和重要性水平确定，通常包括关键业务流程的控制点。

### 问题2：如何处理控制偏差？
**解决方案**：分析偏差原因，评估偏差影响，必要时调整审计策略。

### 问题3：如何选择样本量？
**解决方案**：根据控制频率、重要性水平和可接受偏差率确定样本量。

### 问题4：如何评价控制有效性？
**解决方案**：综合考虑控制设计、执行情况和偏差率等因素。

#### 四、审计说明

**控制测试执行要点：**

1. **现金收付控制测试**
   - 检查现金收付是否经过适当审批
   - 验证现金收付记录的完整性和准确性
   - 确认现金收付程序的合规性

2. **现金盘点控制测试**
   - 观察现金盘点流程的执行
   - 检查盘点记录的完整性和准确性
   - 验证盘点差异的处理程序

3. **银行账户管理控制测试**
   - 检查银行账户开立、变更、撤销的审批程序
   - 验证银行印鉴的分管制度
   - 确认银行账户的合规性

4. **银行对账控制测试**
   - 检查银行对账的及时性和准确性
   - 验证对账差异的调节和处理
   - 确认对账复核程序的执行

5. **资金支付控制测试**
   - 检查资金支付的审批程序
   - 验证支付权限的合理性
   - 确认支付记录的完整性

**重点关注领域：**
- 现金安全管理和盘点制度
- 银行印鉴分管和对账制度
- 资金支付审批权限控制
- 大额资金调拨的特殊审批
- 资金安全监控机制

**常见控制缺陷：**
- 现金盘点不及时或不准确
- 银行对账存在差异未及时处理
- 资金支付审批权限不清晰
- 银行印鉴保管制度不完善
- 资金流向监控不充分
