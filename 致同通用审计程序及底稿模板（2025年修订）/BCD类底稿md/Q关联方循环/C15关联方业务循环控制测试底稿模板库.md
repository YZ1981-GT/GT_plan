# C15 关联方业务循环控制测试底稿模板库

## 📖 文档概述

本文档为关联方业务循环控制测试底稿模板库，包含关联方相关的所有控制测试程序模板，适用于致同会计师事务所的审计工作。

## 🚀 使用说明

1. **控制测试选择**：根据被审计单位的实际情况选择合适的控制测试模板
2. **程序执行**：按照底稿中的控制测试程序逐步执行
3. **记录完整**：确保所有控制测试程序都有完整的执行记录
4. **偏差处理**：及时识别和处理控制偏差

## 📋 标准化选项清单

### 🎯 认定维度选项
- **存在/发生**：确认关联方业务交易和事项确实发生，关联方关系确实存在
- **完整性**：确认所有应当记录的关联方业务交易和事项均已记录
- **准确性、计价和分摊**：确认关联方业务交易和事项以恰当的金额记录
- **权利和义务**：确认企业拥有或控制关联方相关的权利和义务
- **截止**：确认关联方业务交易和事项记录于正确的会计期间
- **分类**：确认关联方业务交易和事项记录于恰当的账户
- **列报**：确认关联方业务在财务报表中得到恰当列报和披露

### ⚙️ 控制属性选项
- **人工的**：完全由人工执行的控制活动
- **自动化的**：由信息系统自动执行的控制活动
- **人工依赖信息系统控制**：依赖信息系统产生信息的人工控制

### 🔍 控制测试方法详细说明
包含询问、检查、观察、重新执行、前期、前推、利用内部审计工作、利用服务机构审计师报告等8种方法的详细说明。

4. **关联方披露控制**：列报 + 完整性

## 📊 关联方循环业务流程图及描述

### 关联方循环主要业务流程

```
关联方识别 → 档案管理 → 交易审批 → 交易执行 → 交易监控 → 信息披露 → 合规审查 → 风险评估 → 风险应对 → 内控监督 → 外部沟通
    ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓
  合规部     档案部     董事会     业务部门    监察部     董秘办     合规部     风控部     风控部     内审部     董秘办
```

### 详细流程描述

**A. 关联方识别阶段**
1. **关联方识别**：识别和确认关联方关系
2. **关联方登记**：建立关联方信息档案
3. **关联方更新**：定期更新关联方信息
4. **关联方确认**：确认关联方关系变化
5. **关联方公告**：公告关联方信息

**B. 关联方交易阶段**
1. **交易识别**：识别关联方交易事项
2. **交易审批**：审批关联方交易
3. **交易执行**：执行关联方交易
4. **交易记录**：记录交易详细信息
5. **交易监控**：监控交易执行情况

**C. 信息披露阶段**
1. **披露识别**：识别需要披露的信息
2. **披露准备**：准备披露文件
3. **披露审核**：审核披露信息
4. **披露执行**：执行信息披露
5. **披露跟踪**：跟踪披露效果

**D. 合规管理阶段**
1. **合规审查**：进行关联方交易合规审查
2. **风险评估**：评估关联方交易风险
3. **风险应对**：制定风险应对措施
4. **内控监督**：监督内控制度执行
5. **外部沟通**：与监管机构沟通

### 关键控制点识别

| 业务环节 | 关键风险 | 控制措施 | 控制类型 |
|----------|----------|----------|----------|
| 关联方识别 | 识别不全、信息滞后 | 定期识别、信息更新、档案管理 | 预防性+检查性 |
| 关联方交易 | 价格不公允、程序不当 | 交易审批、价格评估、程序控制 | 预防性 |
| 信息披露 | 披露不充分、信息错误 | 披露审核、信息核实、及时披露 | 预防性+检查性 |
| 合规管理 | 合规风险、监管处罚 | 合规审查、风险评估、内控监督 | 检查性 |

### 容易出错的领域

1. **关联方识别**：关联方关系识别不全面，信息更新不及时
2. **交易审批**：关联方交易审批程序不规范，独立董事意见缺失
3. **价格公允性**：关联方交易价格公允性评估不充分
4. **信息披露**：关联方信息披露不及时或不充分
5. **合规管理**：关联方交易合规审查不到位，风险防控不足

## ⚠️ 常见问题与解决方案

### 问题1：如何确定控制测试范围？
**解决方案**：根据风险评估结果和重要性水平确定，通常包括关键业务流程的控制点。

### 问题2：如何处理控制偏差？
**解决方案**：分析偏差原因，评估偏差影响，必要时调整审计策略。

### 问题3：如何选择样本量？
**解决方案**：根据控制频率、重要性水平和可接受偏差率确定样本量。

### 问题4：如何评价控制有效性？
**解决方案**：综合考虑控制设计、执行情况和偏差率等因素。

#### 一、审计目标
汇总各流程中经测试发现的控制偏差，并得出结论。

#### 二、控制例外情况描述
经理层监督客户投诉的数量及类型并把这些信息作为估计销售退回的依据。然而，注册会计师在执行时发现某一期间计算出的投诉率没有包含某一地区收到的投诉。

#### 三、控制偏差评价步骤

| 评价步骤 | 是/否 | 评价分析过程 | 备注 |
|----------|-------|-------------|------|
| **步骤一**：确认发现的控制例外情况（选定的样本无法执行控制测试）是否属于控制偏差（内部控制运行偏离设计）？ | **是** | 该控制例外情况影响销售收入的金额，且金额并不关紧要。进入步骤二 | |
| **步骤二**：确定控制偏差的性质：系统性偏差？人为有意造成的偏差？随机性偏差？ | **随机性偏差** | 注册会计师确定这只是众多地区中唯一的一个没有报告投诉率的，而且其他所有月份都完成了投诉率的报告。确定应对措施：进入步骤三 | |
| **步骤三**：对于随机性的偏差，确定适当的审计应对措施是否进行测试？直接将该项控制偏差视作缺陷？ | **扩大样本量** | 注册会计师确定该项控制偏差是随机性的，并应扩大样本量来予以确认。进入步骤四 | |
| **步骤四**：根据发现的控制偏差的数量来确定扩大样本量的范围，扩大样本量进行测试还是否发现新的偏差？ | **否** | 尽管存在一项控制偏差，但测试范围内的错报率在可容忍范围内且该控制偏差不具有代表性。**控制有效** | |
| **步骤五**：评价控制缺陷。 | | | |
| **步骤六**：若确定发现的控制例外情况不属于控制偏差，则可选取替代样本以获得证明控制运行有效的充分、适当的审计证据。但需确定控制例外情况是否表明相关控制的设计存在缺陷？ | | | |

#### 四、评价结论
**该控制例外情况属于随机性偏差，扩大样本测试后未发现偏差，控制有效**

#### 三、控制缺陷方法说明

**询问**：是指向被审计单位的内控负责人员询问内控情况，以获取审计证据来确定内控缺陷。

**检查**：是指对被审计单位内控制度相关文件的检查，以及相关资料、电子文档的检查并获得证据以确定内控缺陷。

**观察**：是指观察被审计单位人员正在执行的内控活动或程序的操作。

**重新执行**：是指审计人员独立执行作为被审计单位内控组成部分的程序或控制。

**利用内部审计工作**：是指在某些情况下，可以利用内部审计人员执行的工作来获得审计证据。

**利用服务机构的审计师工作**：是指利用为服务机构提供审计服务的注册会计师一系统服务机构的控制运行有效性提供合理保证的审计意见。

## 📋 控制测试执行详细注释

### 注1：控制例外情况及控制偏差

**控制测试程序的执行指导：**

对于一个选定的样本项目代表性，如果无法执行控制测试的控制点进行汇总，则应当代表性，例如注册会计师无法对某一个选定的样本无法执行控制测试，则应当考虑：

例如，对于一个选定的样本，如果出于某种不可预见的原因而无法执行控制测试，则应当对控制测试，则应当考虑该情况是否属于控制偏差。

**控制例外情况的分析：**
- （1）根据选定的样本情况，注册会计师确定该例外情况是否为控制偏差，即是否属于控制运行偏离设计的情况，即是否属于控制运行偏离设计的情况。
- （2）基于对控制设计的了解，注册会计师应当确定该控制例外情况是否属于控制偏差，如果属于控制偏差，则应当进一步确定控制偏差的性质。

**控制偏差：**
控制偏差是指控制运行偏离控制设计的情况，是指在合理保证控制目标实现的前提下，控制在某个时点或某个期间内未按设计运行或运行效果未达到预期的情况。

发生控制偏差时应当进行控制测试的控制点进行汇总，注册会计师应当考虑控制偏差对审计策略的影响，以及对控制测试结论的影响。

### 注2：控制偏差的性质

**控制偏差是否属于以下情况：**

通过检查是否属于以下情况，注册会计师可以全面评价控制偏差：

通过性质是否属于偶然发生的偏差，例如控制偏差是否是某一个立即引起的，是指只有某个特定的条件下才会发生，而且其他所有月份都完成了投诉率的报告。

如果发现的控制偏差是否属于某种特定条件下发生的偏差，注册会计师应当考虑控制偏差对控制有效性评价的影响，以及对审计策略的影响。

**控制偏差性质的分类：**
- **系统性偏差**：控制设计本身存在缺陷，导致控制无法有效运行
- **人为有意偏差**：相关人员故意不执行控制或故意绕过控制
- **随机性偏差**：偶然发生的控制执行偏差，不具有代表性

### 注3：随机性偏差的应对措施

**如果认为：**

如果认为控制偏差属于随机性偏差，期间当选择控制测试的控制点进行汇总，例如：

- （1）在检查某一期间计算出的投诉率时发现某一地区收到的投诉，但是其他所有月份都完成了投诉率的报告
- （2）在检查某项控制的执行情况时，发现某个月份的控制执行存在偏差，但其他月份的控制执行都是有效的

**应对措施选择：**
- 扩大样本量进行进一步测试
- 执行替代控制测试程序
- 评价偏差对控制整体有效性的影响

如果确定为随机性偏差，注册会计师应当考虑是否需要扩大样本量，以及是否需要执行额外的控制测试程序。

### 注4：扩大样本量的应对措施

**如果扩大样本量测试范围：**

如果扩大样本量测试后发现偏差，则应当重新评价控制的有效性，例如注册会计师可以再次选择控制测试的控制点进行汇总。

如果扩大样本量测试后未发现新的偏差，则应当考虑：
- 原发现的偏差是否具有代表性
- 控制整体运行是否有效
- 是否需要调整对控制的依赖程度

**样本量扩大的考虑因素：**
- 原样本量的大小
- 发现偏差的性质和频率
- 控制的重要性
- 可接受的偏差率

### 注5：控制偏差的应对措施

**若确定发现的控制例外情况属于控制偏差：**

若确定发现的控制例外情况属于控制偏差，注册会计师应当：

- 评价控制偏差对控制有效性的影响
- 考虑控制偏差对审计策略的影响  
- 确定是否需要修改控制测试的性质、时间和范围
- 考虑是否需要执行额外的实质性程序

**控制偏差对审计的影响：**
- 可能需要降低对控制的依赖程度
- 可能需要扩大实质性程序的范围
- 可能需要调整重要性水平的判断
- 可能影响管理层建议书的内容

**后续审计程序的调整：**
- 增加实质性分析程序的精确度
- 扩大细节测试的样本量
- 调整审计程序的执行时间
- 考虑专家工作的利用

| 如果发现的控制偏差是系统性偏差或人为有意造成的偏差，注册会计师应当考虑是否存在舞弊的迹象以及对审计方案的影响。 |  |  |  |  |  |  |  |  |
| 注册会计师在将控制偏差确认为随机性的时候需要谨慎，在得出该结论之前，应当实施追加的审计程序，以高度确信该偏差 |  |  |  |  |  |  |  |  |
| 对总体误差不具有代表性。 |  |  |  |  |  |  |  |  |
| 在评价控制偏差的性质时，需要考虑： |  |  |  |  |  |  |  |  |
| （1）控制偏差是否具有共同的特征。例如，控制偏差发生在某些特定类型的业务、地点或生产线中，或者发生在同样的人员 |  |  |  |  |  |  |  |  |
| 执行控制时，抑或是发生在某一特定的时期。 |  |  |  |  |  |  |  |  |
| （2）控制偏差是否是由于基础工作错误造成的。例如，控制偏差是由于错误的公司或计算造成的，或者是基于不完整或 |  |  |  |  |  |  |  |  |
| 不准确的数据造成的。 |  |  |  |  |  |  |  |  |
| 如果确认偏差时系统性的或人为有意造成的，应直接将该控制偏差视为一项内部控制缺陷。 |  |  |  |  |  |  |  |  |
| 注3：随机性偏差的应对措施 |  |  |  |  |  |  |  |  |
| 如果认为一项控制偏差是随机的，则应当确定适当的审计应对措施。包括： |  |  |  |  |  |  |  |  |
| （1）扩大样本量进行测试，从而确定测试发现的错误率在可容忍范围内； |  |  |  |  |  |  |  |  |
| （2）直接将该项控制偏差视作一项内部控制缺陷。 |  |  |  |  |  |  |  |  |
| 如果决定扩大样本量进行测试，注册会计师应当根据发现的控制偏差的数量来确定扩大样本量的范围。例如，对于每日发生 |  |  |  |  |  |  |  |  |
| 多次的控制，如果初始样本规模为25个，当测试发现一项控制偏差，且该偏差不是系统性偏差时，注册会计师扩大样本规模 |  |  |  |  |  |  |  |  |
| 进行测试，所增加的样本规模至少为15个。 |  |  |  |  |  |  |  |  |
| 注4：扩大样本量后的测试结果 |  |  |  |  |  |  |  |  |
| 如果扩大样本量测试后再次发现偏差，则注册会计师可以得出该控制无效的结论。 |  |  |  |  |  |  |  |  |
| 如果扩大样本量测试没有再次发现偏差，则注册会计师可以得出该控制有效的结论。即尽管存在一项控制偏差，但测试范围 |  |  |  |  |  |  |  |  |
| 内的错误率在可容忍范围内且该控制偏差不具有代表性。 |  |  |  |  |  |  |  |  |
| 单项控制并非一定要毫无偏差地运行，才被认为有效。 |  |  |  |  |  |  |  |  |
| 注5：评价是否构成设计缺陷 |  |  |  |  |  |  |  |  |
| 若确定发现的控制例外情况不属于控制偏差，则可以选取替代样本以获得证明控制运行有效的充分、适当的审计证据。 |  |  |  |  |  |  |  |  |
| 但需确定控制例外情况是否表明相关控制的设计存在缺陷。 |  |  |  |  |  |  |  |  |
| 设计缺陷是指缺少为实现控制目标所必需的控制，或现有控制设计不适当、即使正常运行也难以实现预期的控制目标。 |  |  |  |  |  |  |  |  |
