# C2 销售循环业务层面控制底稿模板库

## 文档概述

本文档为销售循环业务层面控制测试底稿模板库，包含销售循环相关的所有控制测试程序模板，适用于致同会计师事务所的审计工作。

## 使用说明

1. **控制测试选择**：根据被审计单位的实际情况选择合适的控制测试模板
2. **程序执行**：按照底稿中的控制测试程序逐步执行
3. **记录完整**：确保所有控制测试程序都有完整的执行记录
4. **偏差处理**：及时识别和处理控制偏差

## 📊 销售循环业务流程图及描述

### 销售循环主要业务流程

```
客户询价 → 报价审批 → 合同签订 → 订单录入 → 信用审查 → 发货准备 → 发货执行 → 开具发票 → 收款管理 → 售后服务
    ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓         ↓
  市场部     销售部     法务部     销售部     财务部     仓库部     仓库部     财务部     财务部     客服部
```

### 详细流程描述

**A. 销售合同签订阶段**
1. **客户询价**：客户提出产品或服务需求
2. **报价准备**：销售部门准备报价单，包含产品规格、价格、交期等
3. **报价审批**：销售经理审批报价，大额报价需总经理审批
4. **合同谈判**：与客户就合同条款进行谈判
5. **合同签订**：经法务部审核后签订正式销售合同

**B. 订单处理阶段**
1. **订单录入**：销售人员在系统中录入销售订单
2. **订单审批**：销售经理审批订单，确认价格、数量、交期
3. **信用审查**：财务部门审查客户信用状况和授信额度
4. **库存检查**：仓库部门检查库存是否满足订单需求
5. **生产安排**：如需生产，安排生产计划

**C. 发货执行阶段**
1. **发货准备**：仓库根据订单准备发货
2. **质量检验**：质检部门对产品进行最终检验
3. **发货审批**：仓库主管审批发货单
4. **发货执行**：安排物流发货，更新发货记录
5. **发货确认**：客户确认收货

**D. 开票收款阶段**
1. **开票申请**：销售部门申请开具发票
2. **开票审核**：财务部门审核开票信息
3. **发票开具**：开具正式销售发票
4. **收款跟踪**：财务部门跟踪应收账款回收
5. **收款确认**：确认收款并更新账务记录

### 关键控制点识别

| 业务环节 | 关键风险 | 控制措施 | 控制类型 |
|----------|----------|----------|----------|
| 合同签订 | 合同条款不利、客户信用风险 | 合同审批制度、信用评估 | 预防性 |
| 订单处理 | 订单信息错误、超授信销售 | 订单审批、信用额度控制 | 预防性 |
| 发货执行 | 发货错误、未经授权发货 | 发货审批、三单匹配 | 预防性 |
| 开票收款 | 开票错误、收款延迟 | 开票审核、收款跟踪 | 预防性+检查性 |

### 容易出错的领域

1. **收入确认时点**：发货与开票时间不匹配，影响收入确认准确性
2. **销售折扣处理**：折扣审批不规范，影响收入金额准确性
3. **关联方交易**：关联方销售未按规定审批和披露
4. **期末截止**：期末销售截止测试不充分，影响收入完整性
5. **应收账款管理**：应收账款催收不及时，坏账准备计提不充分

## 📋 标准化选项清单

### 认定维度选项
| 认定类型 | 说明 |
|----------|------|
| 存在/发生 | 记录的交易和事项已经发生，且与被审计单位有关 |
| 完整性 | 所有应当记录的交易和事项均已记录 |
| 准确性、计价和分摊 | 交易和事项以恰当的金额包括在财务报表中，收入或费用被分摊至恰当的期间 |
| 权利和义务 | 被审计单位拥有或控制记录的资产的权利，负债是被审计单位的义务 |
| 截止 | 交易和事项记录于正确的会计期间 |
| 分类 | 交易和事项记录于恰当的账户 |
| 列报 | 交易和事项在财务报表中被恰当列报和披露 |

### 控制属性选项
| 控制属性 | 说明 |
|----------|------|
| 人工的 | 由人工执行的控制活动 |
| 自动化的 | 由信息系统自动执行的控制活动 |
| 人工依赖信息系统控制 | 人工控制依赖于信息系统产生的信息 |

### 频率选项
| 频率类型 | 说明 |
|----------|------|
| 每笔交易 | 对每笔交易都执行的控制 |
| 每天 | 每日执行的控制 |
| 每周 | 每周执行的控制 |
| 每半月 | 每半月执行的控制 |
| 每月 | 每月执行的控制 |
| 每季度 | 每季度执行的控制 |
| 每年 | 每年执行的控制 |
| 非常规/低运行频率 | 不定期或低频率执行的控制 |
| 其他（描述） | 其他特殊频率的控制 |

### 与控制相关的风险等级
| 风险等级 | 说明 |
|----------|------|
| 高 | 控制失效对财务报表产生重大影响的可能性高 |
| 中 | 控制失效对财务报表产生中等程度影响 |
| 低 | 控制失效对财务报表影响相对较小 |

### 控制测试方法详细说明
| 测试方法 | 详细说明 |
|----------|----------|
| 询问 | 是指向被审计单位的内部或外部的知情人员获取财务信息和非财务信息 |
| 检查 | 是指对被审计单位的内部或外部生成的，以纸质、电子或其他介质形式存在的记录和文件进行审查，或对资产进行实物检查 |
| 观察 | 是指察看相关人员正在从事的活动或实施的程序 |
| 重新执行 | 是指独立执行作为被审计单位内部控制组成部分的控制或检测程序 |
| 前期 | 利用以前审计期间获取的审计证据，如果前期审计期间获取的有关控制运行有效性的审计证据 |
| 前推 | 利用期中获取的审计证据，通过实施期中获取或更新审计证据，可能会在期末之前执行控制测试 |
| 利用内部审计工作 | 在某些情况下，可以利用内部审计师的工作来获取控制测试 |
| 利用服务机构的审计师 | 服务注册会计师向第三方报告可能为与服务组织控制的运行有效性提供相关审计证据 |

## 📋 标准化选项使用指南

### 如何使用认定维度选项
1. **选择相关认定**：根据控制点的性质选择1-3个最相关的认定
2. **常见组合**：
   - 销售合同控制：存在/发生 + 权利和义务
   - 收入确认控制：存在/发生 + 完整性 + 截止
   - 应收账款控制：存在/发生 + 准确性 + 权利和义务
   - 销售价格控制：准确性 + 权利和义务
   - 发货控制：存在/发生 + 完整性 + 截止

### 如何确定控制属性
- **人工的**：需要人工判断和执行的控制（如合同审批、价格审核）
- **自动化的**：由系统自动执行的控制（如系统权限控制、自动计价）
- **人工依赖信息系统控制**：人工控制但依赖系统信息（如应收账款管理）

### 如何选择测试方法
- **检查**：适用于有书面证据的控制（如合同审批单据、发票）
- **观察**：适用于现场执行的控制（如发货流程、开票过程）
- **询问**：适用于了解控制执行情况和政策理解
- **重新执行**：适用于验证控制有效性（如重新计算价格、复核账龄）

### 如何确定样本量
参考**控制测试抽样指引表**：
- 根据控制频率确定基础样本量
- 考虑控制重要性和风险等级调整
- 高风险控制（如收入确认、大额销售）适当增加样本量

## ⚠️ 常见问题与解决方案

### 问题1：如何确定控制测试范围？
**解决方案**：根据风险评估结果和重要性水平确定，通常包括关键业务流程的控制点。

### 问题2：如何处理控制偏差？
**解决方案**：分析偏差原因，评估偏差影响，必要时调整审计策略。

### 问题3：如何选择样本量？
**解决方案**：根据控制频率、重要性水平和可接受偏差率确定样本量。

### 问题4：如何评价控制有效性？
**解决方案**：综合考虑控制设计、执行情况和偏差率等因素。

#### 四、审计说明

**控制测试执行要点：**

1. **销售合同签订控制测试**
   - 检查销售合同是否经过适当审批
   - 验证合同条款的完整性和准确性
   - 确认客户信用评估程序的执行

2. **销售订单审批控制测试**
   - 检查销售订单审批权限矩阵
   - 验证超限额订单的特殊审批程序
   - 确认订单与合同的一致性

3. **发货控制测试**
   - 观察发货流程的执行
   - 检查发货单与销售订单的匹配
   - 验证发货授权和记录的完整性

4. **开票控制测试**
   - 检查发票开具的及时性和准确性
   - 验证发票与发货单的匹配
   - 确认税务处理的正确性

5. **收款控制测试**
   - 检查收款记录的完整性
   - 验证收款与销售的匹配
   - 确认坏账准备计提的合理性

**重点关注领域：**
- 收入确认时点的准确性
- 销售折扣和返利的处理
- 关联方销售的特殊审批
- 大额销售的特别关注
- 期末销售截止测试

**常见控制缺陷：**
- 销售合同审批流程不完整
- 信用额度控制执行不到位
- 发货与开票时间差异过大
- 收款记录不及时更新
- 销售退回处理程序不规范

#### 四、控制偏差评价步骤

**控制例外情况描述：**
经理层监督客户投诉的数量及类型并把这些信息作为估计销售退回的依据。然而，注册会计师在执行控制测试时发现某一期间计算出的投诉率没有包含某一地区收到的投诉。

**评价步骤：**

| 评价步骤 | 是否 | 评价分析过程 | 备注 |
|----------|------|-------------|------|
| **步骤一：** 确认发现的控制例外情况（选定的样本无法执行控制测试）是否属于控制偏差（内控制运行偏离设计）？ | 是 | 该控制例外情况影响销售收入的金额，且金额不属于无关紧要。<br><br>进入步骤二 | |
| **步骤二：** 确定控制偏差的性质：系统性偏差？人为有意造成的偏差？随机性偏差？ | 随机性偏差 | 注册会计师确定这只是众多地区中唯一一个没有报告投诉率的，而且其他所有月份都完成了投诉率的报告。<br><br>确定应对措施：进入步骤三 | |
| **步骤三：** 对于随机性的偏差，确定适当的审计应对措施是否需要进行测试？直接将该项控制偏差视作缺陷？ | 扩大样本量 | 注册会计师确定该项控制偏差是随机性的，并应扩大样本量来予以确认。<br><br>进入步骤四 | |
| **步骤四：** 根据发现的控制偏差的数量来确定扩大样本量的范围，扩大样本量进行测试是否发现新的偏差？ | 否 | 尽管存在一项控制偏差，但测试范围内的错报率在可容忍范围内且该控制偏差不具有代表性。<br><br>控制有效 | |
| **步骤五：** 评价控制缺陷。 | | | |
| **步骤六：** 若确定发现的控制例外情况不属于控制偏差，则可以选取替代样本以获得证明控制运行有效的充分、适当的审计证据。但需确定控制例外情况是否表明相关控制的设计存在缺陷？ | | | |

**评价结论：**
该控制例外情况属于随机性偏差，扩大样本量测试后未发现偏差，控制有效
