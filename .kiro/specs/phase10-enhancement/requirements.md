# Phase 10 — 深度增强与协作体验

## 概述
Phase 9 完成了前后端联动集成。Phase 10 聚焦于生产环境深度增强：底稿全生命周期管理（下载/导入/归档/溯源）、LLM 深度融合（对话填充底稿/复核/分析）、多人协作体验（复核对话/推送联动/工时打卡）、合并报表增强（锁定同步/附注明细/独立模块）、服务器存储与缓存管理。

## 需求清单

### 需求 1：底稿下载与导入（服务器部署核心）

#### 1a 自定义选表与底稿下载
- 1a.1 用户可按审计循环/科目/底稿类型自定义勾选需要下载的报表和底稿
- 1a.2 勾选后一键打包下载为 ZIP（含目录结构：循环/底稿编号_名称.xlsx）
- 1a.3 下载时自动预填最新四表数据到底稿（调用 PrefillService）
- 1a.4 支持单个底稿下载和批量下载两种模式

#### 1b 底稿导入（离线编辑后回传）
- 1b.1 用户在本地 Excel 编辑完底稿后，上传回系统
- 1b.2 上传时自动检测版本冲突（file_version 比对）
- 1b.3 冲突时提示用户选择：覆盖服务器版本 / 保留服务器版本 / 手动合并
- 1b.4 导入后自动触发 ParseService 解析关键单元格，回写 parsed_data
- 1b.5 导入后触发 WORKPAPER_SAVED 事件，级联更新试算表和报表

### 需求 2：连续审计与跨年联动

#### 2a 连续审计项目推送
- 2a.1 上年项目归档后，一键创建当年项目（继承基本信息/科目映射/模板集/团队委派）
- 2a.2 上年底稿的审定数自动推送为当年底稿的期初数
- 2a.3 上年附注的期末余额自动推送为当年附注的上期金额
- 2a.4 上年调整分录中标记为"连续"的分录自动结转到当年
- 2a.5 上年未更正错报自动结转到当年（carry_forward）

#### 2b 跨年数据对比
- 2b.1 当年底稿中可查看上年同科目底稿数据（PREV 函数已支持）
- 2b.2 附注变动分析自动对比上年数据生成变动说明

### 需求 3：服务器存储与分区管理

#### 3a 项目级分区存储
- 3a.1 底稿文件按 storage/projects/{project_id}/{year}/workpapers/ 分区存储（已实现）
- 3a.2 附件按 storage/projects/{project_id}/attachments/ 分区存储
- 3a.3 用户私人库按 storage/users/{user_id}/private/ 分区存储
- 3a.4 每个用户私人库容量上限（默认 1GB），达到 90% 时提示清理
- 3a.5 存储统计看板：按项目/用户/年度展示存储占用

#### 3b 底稿归档联动
- 3b.1 项目归档时，所有底稿状态锁定为 archived（只读）
- 3b.2 归档后底稿文件压缩为 .tar.gz 移到冷存储（wp_storage_service 已实现）
- 3b.3 归档后自动生成归档清单（底稿编号/名称/状态/编制人/复核人/版本）
- 3b.4 归档后触发人员简历自动丰富（enrich_resume 已实现）

### 需求 4：过程记录与附件关联

#### 4a 全程操作留痕
- 4a.1 所有关键操作记录到 logs 表（已有 AuditLogMiddleware）
- 4a.2 底稿编辑记录：谁在什么时间编辑了哪个底稿的哪些单元格
- 4a.3 复核记录：复核意见/回复/解决的完整对话链
- 4a.4 调整分录记录：创建/修改/审批的完整流程
- 4a.5 人机协同标注：AI 生成的内容标记为"AI 辅助"，人工确认后标记为"已确认"

#### 4b 附件与底稿关联
- 4b.1 每个底稿可关联多个附件（合同/银行对账单/函证回函等）
- 4b.2 附件上传时自动关联到当前底稿（attachment_working_paper 关联表已有）
- 4b.3 底稿中可查看关联附件列表，点击预览/下载
- 4b.4 附件支持双向溯源：从附件查看关联的底稿，从底稿查看关联的附件

### 需求 5：LLM 深度融合底稿编制

#### 5a 对话式底稿填充
- 5a.1 在底稿编辑页面打开 LLM 对话窗口，对话内容可直接填充到底稿指定单元格
- 5a.2 LLM 根据四表数据+知识库内容+历史底稿，生成底稿内容建议
- 5a.3 支持"帮我填写审定表"/"分析本科目变动原因"/"生成审计说明"等指令
- 5a.4 对话窗口与底稿编辑器联动：选中单元格后对话，AI 回复自动填入

#### 5b 台账分析底稿自动生成
- 5b.1 LLM 根据序时账/辅助明细自动生成台账分析底稿（大额交易/异常交易/关联方交易）
- 5b.2 自动生成审计说明（变动分析/风险提示/审计结论）
- 5b.3 生成的底稿保存到项目底稿目录，自动创建 wp_index 记录

#### 5c 知识库索引调用
- 5c.1 LLM 对话时可引用知识库中的具体文件夹/文档内容
- 5c.2 底稿编辑时可搜索知识库，将搜索结果插入底稿
- 5c.3 知识库内容与底稿双向关联（从底稿查看引用的知识库文档，从知识库查看被哪些底稿引用）

### 需求 6：抽样程序增强

- 6.1 抽样配置与执行（SamplingService 已有基础，Phase 1b 已实现）
- 6.2 抽样结果与底稿关联：抽样选中的样本自动填入对应底稿
- 6.3 MUS 评价结果自动生成审计结论
- 6.4 截止性测试：自动从序时账提取期末前后 N 天的交易，填入截止性测试底稿
- 6.5 账龄分析：用户自定义账龄区间段（如1年以内/1-2年/2-3年/3年以上），按先进先出（FIFO）原则从账簿记录自动计算各辅助维度的账龄分布，支持直接上传已有账龄表作为兜底
- 6.6 各月明细填充：按月汇总序时账数据，自动填入月度明细分析底稿

### 需求 7：合并报表增强

#### 7a 合并与单体锁定同步
- 7a.1 合并项目调用单体数据时，单体试算表自动锁定（防止合并过程中单体数据变更）
- 7a.2 锁定期间单体项目显示"合并锁定中"提示，禁止修改试算表和调整分录
- 7a.3 合并完成后自动解锁单体
- 7a.4 合并试算表更新时自动同步最新单体数据

#### 7b 合并与单体附注明细
- 7b.1 合并附注中每个科目可展开查看各单体的明细数据（9k.4.5 已设计）
- 7b.2 单体附注变更时自动标记合并附注需刷新（9k.4.3 已设计）
- 7b.3 合并附注支持导入其他审计师审计的单位的报表和附注（非本系统项目）
- 7b.4 导入格式：Excel 报表 + Word 附注，LLM 辅助解析结构化

#### 7c 独立模块使用
- 7c.1 仅使用合并模块：不需要单体项目，直接导入各单位报表数据进行合并
- 7c.2 仅使用报告复核模块：上传审计报告+报表+附注，进行复核校验
- 7c.3 仅使用报告排版模块：上传内容，按致同规范排版导出 Word/PDF

### 需求 8：底稿复核与 LLM 辅助

#### 8a 底稿复核提示词
- 8a.1 每个审计循环配置专属复核提示词（复用 TSJ/ 提示词库）
- 8a.2 复核时 LLM 根据提示词自动检查底稿内容（数据完整性/逻辑一致性/格式规范）
- 8a.3 LLM 复核结果生成结构化 findings（类型/严重度/位置/建议）

#### 8b 复核记录对话联动
- 8b.1 高级别人员（合伙人/项目经理）复核后，保存复核记录
- 8b.2 复核记录可指定发送给项目团队中的指定人员
- 8b.3 被指定人员收到推送通知，可直接与发起人进行对话互动
- 8b.4 对话支持文字/图片/文件附件
- 8b.5 对话由发起方点击"结束"关闭，结束后可导出对话记录
- 8b.6 进行中的对话在看板中显示（待处理复核对话数）
- 8b.7 预留质量控制复核人员的对话反馈通道

#### 8c 导出功能增强
- 8c.1 复核记录导出为 Word/PDF
- 8c.2 对话记录导出为 Word/PDF
- 8c.3 底稿复核报告导出（含所有 findings + 处理状态）

### 需求 9：报告复核溯源

- 9.1 报告复核中点击附注科目，可反溯到对应底稿
- 9.2 溯源展示：该科目的主要明细变动原因 + 底稿中的审计说明
- 9.3 溯源链：报告 → 附注 → 底稿审定表 → 序时账明细
- 9.4 复核人可在溯源页面直接添加复核意见

### 需求 10：工时打卡与足迹

#### 10a 打卡签到
- 10a.1 团队成员每日打卡签到（记录时间+GPS 位置）
- 10a.2 打卡与工时填报联动（打卡时自动创建当日工时草稿）
- 10a.3 打卡记录可视化（日历视图+地图标记）

#### 10b 足迹与工作总结
- 10b.1 每半年自动生成成员足迹报告（去过的城市/客户/项目）
- 10b.2 LLM 自动生成工作总结（参与项目数/工时统计/行业经验/成长轨迹）
- 10b.3 足迹报告推送给成员本人和所属合伙人

### 需求 11：吐槽与求助专栏

- 11.1 团队成员可匿名或实名发布吐槽/求助帖子
- 11.2 支持文字+图片+表情
- 11.3 其他成员可回复互动（评论+点赞）
- 11.4 项目经理/合伙人可查看团队动态
- 11.5 敏感内容过滤（LLM 辅助审核）

### 需求 12：私人库与 LLM 对话

#### 12a 私人库管理
- 12a.1 每个用户有独立的私人库空间（storage/users/{user_id}/private/）
- 12a.2 支持上传文档（Word/Excel/PDF/图片）
- 12a.3 上传后自动 LLM 识别缓存（向量化索引），缓存上限每人 1GB
- 12a.4 达到 90% 容量时提示用户清理
- 12a.5 新生成的底稿/分析报告可指定保存到私人库
- 12a.6 私人库入口放在最左侧第一栏导航中（ThreeColumnLayout navItems），与项目/知识库同级，方便用户随时调用

#### 12b 私人库 LLM 对话
- 12b.1 基于私人库文档的 LLM 对话（RAG 检索增强）
- 12b.2 对话记录支持复制/粘贴/导出 Word
- 12b.3 对话中引用的文档自动标注来源
- 12b.4 私人库与项目知识库打通（可将私人库文档共享到项目知识库）

### 需求 13：辅助余额表明细汇总与可视化

- 13.1 辅助余额表按科目汇总，与科目余额表自动匹配
- 13.2 匹配一致时：只显示汇总行，折叠辅助明细
- 13.3 匹配不一致时：同时显示科目余额表行和辅助汇总行，红色高亮差异
- 13.4 差异提示：显示差异金额和可能原因
- 13.5 可视化：饼图/柱状图展示辅助维度构成（如应收账款按客户分布）

### 需求 14：权限精细化

- 14.1 项目成员只能删除自己创建的内容（底稿/调整分录/附件）
- 14.2 项目经理可删除项目内所有内容
- 14.3 合伙人/管理员可删除任何内容
- 14.4 删除操作记录到审计日志，支持追溯

### 需求 15：单元格级复核批注

- 15.1 报表/附注/底稿的任意单元格可添加复核批注（右键菜单或悬浮按钮）
- 15.2 批注内容：文字+@提及人员+优先级（高/中/低）
- 15.3 批注自动穿透关联：附注单元格批注→底稿对应单元格可见，反之亦然
- 15.4 批注状态流转：待处理→已回复→已解决
- 15.5 与需求 8 的复核对话系统打通：批注可升级为对话（点击"发起对话"）
- 15.6 批注汇总面板：按状态/优先级/科目筛选所有批注

### 需求 16：合并数据快照

- 16.1 合并报表生成时，自动保存各单体试算表的快照（consol_snapshots 表）
- 16.2 快照包含：单体 trial_balance 全量数据 + 抵消分录 + 合并调整
- 16.3 单体数据变更后，合并报表可对比快照差异（哪些科目变了、变了多少）
- 16.4 支持基于历史快照重新生成合并报表（回溯审计轨迹）
- 16.5 快照列表可查看（时间/触发原因/差异摘要）

### 需求 17：底稿智能推荐

- 17.1 根据项目风险评估结果（重要性水平/行业特征/历史审计发现），LLM 推荐重点底稿
- 17.2 推荐结果与程序裁剪联动：高风险科目自动标记为"必须执行"
- 17.3 低风险科目建议简化程序（跳过细节测试，仅做分析性复核）
- 17.4 推荐理由可追溯（LLM 给出推荐依据）

### 需求 18：知识库上下文感知增强

- 18.1 LLM 对话时自动注入当前底稿上下文（科目编码/审计循环/底稿类型/试算表数据）
- 18.2 知识库检索按相关度排序，优先返回同行业/同科目的历史底稿和审计经验
- 18.3 对话中引用知识库内容时，自动标注来源文档和页码
- 18.4 支持"@知识库"触发精准检索（类似聊天中的 @提及）

### 需求 19：年度差异分析报告

- 19.1 自动生成本年 vs 上年的全科目差异分析报告
- 19.2 按变动率排序，高亮重大变动科目（变动率>20% 或变动额>重要性水平）
- 19.3 LLM 自动生成每个重大变动科目的分析说明
- 19.4 报告可导出 Word，作为分析性复核底稿（自动创建 wp_index 记录）
- 19.5 与连续审计联动：上年项目数据自动作为对比基准

### 需求 20：附件智能分类

- 20.1 上传附件时 LLM 自动识别附件类型（合同/发票/银行对账单/函证回函/会议纪要/其他）
- 20.2 自动建议关联到哪个底稿（根据附件内容和底稿科目匹配）
- 20.3 附件内容 OCR 提取后可被 LLM 对话引用
- 20.4 附件分类统计：按类型/科目/上传人展示

### 需求 21：报告排版模板管理

- 21.1 报告排版模板管理（致同标准版/简化版/国企版/上市版）
- 21.2 模板支持自定义（字体/页边距/页眉页脚/水印/Logo）
- 21.3 排版实时预览（iframe 渲染仿 Word 效果）
- 21.4 模板版本管理（修改后自动递增版本号，支持回滚）
- 21.5 模板共享：事务所级模板库，各项目可选用
