# 业务流程端到端联调 — 需求文档

> 版本：v1.0
> 日期：2026-05-13
> 定位：确保"导入→映射→试算表→报表→底稿→附注"完整链路每一步都能在前端正确操作和展示

---

## 前言

### 业务背景

审计作业平台的核心价值链是：**导入企业账套 → 科目映射 → 生成试算表 → 生成财务报表 → 编制底稿 → 编写附注**。每个环节的输出是下一环节的输入，形成闭环。

### 当前状态

- 后端 API 全部存在（151 路由 / 226 服务）
- Seed 数据已全部加载（报表 22 套/1191 行 + 底稿 363 个 + 附注 SOE/Listed + 标准科目 166 个）
- 4 个项目有真实导入数据（陕西华氏/辽宁卫生/四川物流/宜宾大药房）
- auto-match 已验证通过（812 科目 100% 匹配率）
- recalc 直接调用成功（HTTP 层 500 是 uvicorn --reload 问题，非代码 bug）

### 核心问题

**后端能力就绪但前端体验断裂**——用户无法在浏览器中顺畅走完整个流程。具体表现：
1. 试算表重算后前端不刷新（需手动 F5）
2. 报表生成后页面空白（可能缺少 report_line_mapping）
3. 底稿生成入口不明确（用户不知道在哪里触发）
4. 附注生成后表格数据为空（可能缺少 note_wp_mapping）
5. 各环节之间缺少"下一步"引导

### 本 spec 边界

**必做**：确保 4 个真实项目能在浏览器中走完全流程，每步有数据、有展示、有引导。
**不做**：新功能开发、性能优化、多租户、移动端。

## §2 功能需求

### §2.A 试算表环节（F1-F2）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| F1 | recalc 不再 500 | 前端点"全量重算"后 3 秒内返回成功，试算表数据刷新 |
| F2 | recalc 后自动刷新 | 不需要手动 F5，表格数据立即更新 |

### §2.B 报表环节（F3-F5, F15）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| F3 | report_line_mapping 自动就绪 | 生成报表前检查映射，缺失则从 soe_listed_mapping_preset.json 自动加载 |
| F4 | 四张报表正确展示 | ReportView 切换资产负债表/利润表/现金流量表/权益变动表，每张有数据行 |
| F5 | 穿透查询 | 点击报表行次金额，能看到对应的试算表科目明细 |
| F15 | 公式执行 | report_config 中的 TB('1001','audited') 类公式能正确从试算表取值 |

### §2.C 底稿环节（F6-F8）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| F6 | 底稿生成 | 从模板集选择后一键生成项目底稿（≥20 个底稿） |
| F7 | 底稿列表展示 | WorkpaperList 显示底稿编码/名称/循环/状态/负责人 |
| F8 | 科目关联 | 每个底稿关联到对应的标准科目编码（从 wp_account_mapping.json 读取） |

### §2.D 附注环节（F9-F11）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| F9 | 附注生成 | 选择模板类型（SOE/Listed）后生成附注章节目录 |
| F10 | 附注展示 | DisclosureEditor 左侧目录树 + 右侧章节内容（表格/文本） |
| F11 | 自动取数 | 附注表格中"本期金额"列从试算表审定数自动填充 |

### §2.E 流程引导（F12-F13）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| F12 | 下一步引导 | 试算表完成→"生成报表"按钮；报表完成→"生成底稿"；底稿完成→"生成附注" |
| F13 | 科目映射页面 | 独立路由 /projects/:id/mapping，展示映射列表+完成率+手动调整入口 |

### §2.F 基础数据就绪（F14）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| F14 | 标准科目加载 | 启动时或首次 auto-match 时确保 166 个标准科目在 account_chart 中 |

## §6 术语表

| 术语 | 含义 |
|------|------|
| 试算表 | trial_balance 表，四列结构（未审数/RJE/AJE/审定数） |
| 报表 | financial_reports 表，按 report_config 行次结构生成 |
| 底稿 | working_papers 表，从模板生成的审计工作底稿 |
| 附注 | disclosure_notes 表，财务报表附注章节 |
| report_line_mapping | 报表行次 → 标准科目编码的映射关系 |
| wp_account_mapping | 底稿编码 → 标准科目 → 报表行次 → 附注章节的四级映射 |
