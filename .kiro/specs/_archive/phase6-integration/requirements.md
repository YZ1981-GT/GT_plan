### 需求 9：审计底稿深度集成 — 方案 C 混合架构（高优先级）

#### 9a 底稿模板智能初始化
- 9a.1 扫描致同模板文件夹（600+ xlsx/docx），解析文件名提取底稿编号/名称/审计循环，写入 wp_index 表
- 9a.2 项目创建时按选择的模板集，自动从模板文件夹复制文件到项目目录（storage/projects/{id}/workpapers/）
- 9a.3 每个复制的文件自动创建 working_paper 记录，关联 wp_index
- 9a.4 支持增量更新：模板版本升级后，已有项目可选择性更新未编辑的底稿

#### 9b 底稿在线编辑（ONLYOFFICE + WOPI）
- 9b.1 点击底稿 → ONLYOFFICE Document Server 在线编辑（WOPI 协议，已实现）
- 9b.2 audit-formula 插件提供 TB/WP/AUX/PREV/SUM_TB 取数函数（已有）
- 9b.3 保存时 WOPI PutFile 回写到项目目录
- 9b.4 多人在线协作编辑：ONLYOFFICE 原生支持多人同时编辑同一文档，实时光标和修改可见
- 9b.5 编辑锁机制：同一底稿同时编辑人数上限（建议 ≤5 人），超限时排队等待或只读模式
- 9b.6 离线编辑支持：下载底稿 → 本地 Excel 编辑 → 上传回系统（已有冲突检测 detect_conflicts）

#### 9c 数据预填与回写
- 9c.1 打开底稿前，后端用 openpyxl 预填四表数据到 Excel 指定单元格（审定表：未审数/调整/审定数；明细表：期初/期末/发生额）
- 9c.2 保存后触发后端解析（openpyxl 读取关键单元格），审定数/差异金额等回写到 working_paper.parsed_data JSONB
- 9c.3 不打开 ONLYOFFICE 也能在底稿列表中看到关键数字摘要（轻量级 Web 预览）

#### 9g 大数据量与性能方案
- 9g.1 文件存储：底稿文件存本地磁盘（storage/），不存数据库，数据库只存元数据和路径
- 9g.2 大文件处理：超过 10MB 的 Excel 文件，ONLYOFFICE 打开时启用流式加载
- 9g.3 并发控制：WOPI Lock/Unlock 机制防止写冲突（已实现内存锁）
- 9g.4 文件版本管理：每次保存递增 file_version，保留历史版本（最近 N 个）
- 9g.5 存储清理策略：归档项目的底稿文件可压缩归档到冷存储
- 9g.6 预填性能：大批量预填时用后台任务队列（避免阻塞请求），预填完成后通知用户
- 9g.7 索引性能：600+ 底稿的索引树用前端虚拟滚动或分页加载

#### 9h 四表与底稿数据联动（事件驱动）
- 9h.1 四表数据变更时底稿自动刷新：重新上传四表/新增调整分录/试算表重算时，标记受影响底稿的预填数据为"已过期"，底稿列表显示刷新提示
- 9h.2 底稿中的调整分录回写：审计师在底稿（审定表）中发现需调整的金额，可直接从底稿创建调整分录回写到 adjustments 表（不需要手动去调整分录模块重复录入）
- 9h.3 底稿审定数与试算表双向校验：底稿审定数应与试算表审定数一致，不一致时在底稿列表和试算表中都显示差异提示
- 9h.4 底稿→试算表→报表完整数据链：底稿审定数 → 试算表 → 报表行次，任何一环变更都能追溯影响范围
- 9h.5 事件驱动机制：复用已有 EventBus（DATA_IMPORTED/ADJUSTMENT_CREATED/TRIAL_BALANCE_UPDATED），新增 WORKPAPER_SAVED 事件，触发关联数据的级联更新
- 9h.6 调整分录到底稿的反向联动：在调整分录模块创建/修改 AJE/RJE 后，自动标记关联科目的底稿审定表需要刷新，前端底稿列表显示"有新调整"提示

#### 9i 审计程序裁剪与委派（高优先级）
- 9i.1 程序裁剪入口：项目负责人在底稿管理页面，按审计循环（如 D 收入循环）查看预设的全量审计程序列表
- 9i.2 裁剪操作：对每个审计程序步骤可选择"执行/跳过/不适用"，跳过时需填写裁剪理由（如"风险等级低，不执行细节测试"）
- 9i.3 自定义程序：支持在预设程序基础上新增自定义审计程序步骤（插入到指定位置）
- 9i.4 裁剪后的程序与底稿关联：裁剪后只保留需执行的程序步骤，每个步骤关联到具体底稿
- 9i.5 委派到人：裁剪完成后，将该循环的程序+底稿委派给指定成员，成员只看到裁剪后的程序清单
- 9i.6 成员视角：被委派成员打开项目后，看到的是裁剪后的程序列表（不是全量模板），逐步完成并标记状态
- 9i.7 参照其他单位程序：裁剪时可参照已裁剪过的其他单位的同科目程序
  - 弹窗选择参照单位 → 加载该单位该科目的裁剪结果 → 自动应用到当前单位
  - 应用后支持二次编辑（增删改个别程序步骤）
- 9i.8 批量裁剪：集团审计场景下，选择多个子公司 → 批量应用同一套裁剪方案
  - 适用于同业子公司（业务类型相同，审计程序基本一致）
  - 批量应用后每个子公司仍可单独微调
- 9i.9 裁剪方案保存与复用：每次裁剪结果保存为方案（项目+科目+裁剪内容），后续项目可直接引用历史方案

- 9j.4.4 附注编辑页显示数据来源标签：每个自动取数的单元格旁显示来源（如"来自试算表 1001 审定数"），点击可穿透

##### 9j.5 底稿已审数据与全链路同步
- 9j.5.1 底稿审定表中的审定数 = 试算表审定数（双向校验，9h.3 已定义）
- 9j.5.2 底稿明细表中的期末余额应与附注对应表格的合计行一致
- 9j.5.3 底稿中修改审定数 → 触发 WORKPAPER_SAVED → 解析回写 → 与试算表比对 → 差异提示（不自动覆盖试算表，需用户确认）

#### 9k 附注与底稿深度联动 + 模版驱动 + 单体/合并附注联动（高优先级）

##### 9k.1 附注数据从底稿提数
- 9k.1.1 附注表格数据优先从底稿 parsed_data 提取（底稿是第一手审计证据），试算表作为兜底数据源
- 9k.1.2 提数优先级：底稿审定表 parsed_data → 试算表 audited_* → 手动编辑值
- 9k.1.3 每个附注章节关联到具体底稿编号（如"五、9 固定资产"关联 E9-1 固定资产审定表）
- 9k.1.4 底稿中的明细数据（如固定资产分类明细：房屋/机器/运输/电子）直接映射到附注表格行
- 9k.1.5 底稿中的变动数据（期初+本期增加-本期减少=期末）直接映射到附注变动表
- 9k.1.6 提数映射规则配置：每个附注章节定义从哪个底稿的哪些单元格取数（mapping_rules JSONB）

##### 9k.2 附注手动编辑与锁定
- 9k.2.1 自动提数的单元格标记为"自动"（蓝色背景），手动编辑的标记为"手动"（白色背景）
- 9k.2.2 用户可将自动提数的单元格切换为手动模式（解除与底稿的绑定），手动输入值
- 9k.2.3 手动编辑的单元格在底稿数据更新时不被覆盖（锁定机制）
- 9k.2.4 叙述性附注（如或有事项、承诺事项、日后事项）完全由用户手动编辑
- 9k.2.5 会计政策章节从模版预填标准文本，用户可编辑修改
- 9k.2.6 每个单元格支持添加审计批注（审计师对数据的说明或疑问）

##### 9k.3 附注模版驱动（国企版/上市版）
- 9k.3.1 项目创建时选择附注模版类型（国企版 soe / 上市版 listed），决定附注章节结构
- 9k.3.2 国企版（40节）：含国有资本经营信息、国有资产保值增值等特有章节
- 9k.3.3 上市版（45节）：含每股收益、分部报告、股份支付等特有章节，披露要求更详细
- 9k.3.4 模版定义每个章节的：表格结构（headers/rows）、校验预设（check_presets）、宽表公式（wide_table_presets）、正文模版（text_template）
- 9k.3.5 模版可自定义：在标准模版基础上增删章节、修改表格结构（NoteTemplateService 已有 CRUD）
- 9k.3.6 模版版本管理：修改后自动递增版本号，支持回滚到历史版本
- 9k.3.7 附注生成时按模版结构初始化所有章节，自动提数填充表格数据

##### 9k.4 单体附注与合并附注联动
- 9k.4.1 合并附注 = 单体附注汇总 + 合并特有章节（合并范围/商誉/少数股东/内部交易/外币折算）
- 9k.4.2 合并附注的科目数据从各子公司单体附注汇总（不是从合并试算表重新取数）
  - 例：合并附注"固定资产"= 子公司A固定资产 + 子公司B固定资产 - 内部交易抵消
- 9k.4.3 子公司单体附注变更时，自动标记合并附注对应章节为"需刷新"
- 9k.4.4 合并附注刷新时，重新汇总各子公司数据 + 重新计算抵消影响
- 9k.4.5 合并附注编辑页面支持查看各子公司的明细构成（展开行显示子公司拆分）
- 9k.4.6 单体附注与合并附注的章节编号体系保持一致（五、1 货币资金 → 合并五、1 货币资金）

##### 9k.5 附注与报表交叉引用
- 9k.5.1 报表行次自动关联附注章节编号（如资产负债表"货币资金"行 → 附注五、1）
- 9k.5.2 附注章节标题旁显示对应报表行次金额，不一致时红色警告
- 9k.5.3 附注合计行金额应与报表行次金额一致（已有 NoteValidationEngine.validate_balance）
- 9k.5.4 附注编辑页面侧边栏显示报表对应行次数据，方便对照

##### 9k.6 附注与底稿双向穿透
- 9k.6.1 附注表格 → 点击数据来源标签 → 打开对应底稿（ONLYOFFICE 编辑器）
- 9k.6.2 底稿审定表 → 查看关联附注 → 跳转到附注编辑页对应章节
- 9k.6.3 底稿明细表数据变更 → 附注对应行自动刷新（如果是自动提数模式）
- 9k.6.4 附注手动修改金额 → 与底稿数据不一致时显示差异提示

##### 9k.7 附注导出
- 9k.7.1 附注导出为 Word 文档（复用已有 Word 导出排版规范）
- 9k.7.2 导出时按模版结构生成完整附注（含表格+叙述性文字+页码）
- 9k.7.3 合并附注导出包含合并特有章节
- 9k.7.4 支持选择性导出（勾选需要导出的章节）

#### 9l 附注编辑体验 + 历史附注复用 + LLM 辅助（高优先级）

##### 9l.1 附注编辑方案 — 结构化 HTML 编辑器（非 Excel/Word）
- 9l.1.1 附注编辑采用内置结构化 HTML 编辑器，不用 Excel 或 Word（理由：附注有严格的章节结构+表格+叙述文字混排，Excel 无法处理叙述文字，Word 编辑体验差且难以提数联动）
- 9l.1.2 表格部分：el-table 可编辑表格（单元格双击编辑），自动提数单元格蓝色标识，手动单元格白色
- 9l.1.3 叙述文字部分：TipTap 富文本编辑器（支持标题/段落/列表/加粗/斜体），每个章节的叙述区域独立
- 9l.1.4 混排布局：每个附注章节 = 章节标题 + 叙述文字区（可选）+ 表格区（可选）+ 补充说明区（可选）
- 9l.1.5 编辑器内所有内容存储为结构化 JSON（table_data + text_blocks），导出时按模版格式渲染为 Word
- 9l.1.6 实时预览：右侧面板显示 Word 导出效果预览（仿宋_GB2312 字体、标准页边距、表格边框样式）

##### 9l.2 附注章节裁剪（参照审计程序裁剪逻辑）
- 9l.2.1 附注模版包含全量章节（国企版40节/上市版45节），项目负责人可裁剪不适用的章节
- 9l.2.2 裁剪操作：每个章节可选择"保留/跳过/不适用"，跳过时需填写理由
- 9l.2.3 裁剪后的章节列表 = 用户实际需要编辑的附注内容
- 9l.2.4 裁剪方案保存与复用：保存为方案（项目+模版类型+裁剪内容），后续项目可参照
- 9l.2.5 参照其他单位附注裁剪：弹窗选择参照项目 → 加载裁剪结果 → 应用到当前项目
- 9l.2.6 批量裁剪：集团审计场景下，选择多个子公司批量应用同一套裁剪方案

##### 9l.3 上传历史附注 — 智能解析与复用
- 9l.3.1 支持上传上年或以往年度附注文件（Word .docx / PDF 含图片和图层）
- 9l.3.2 Word 解析：python-docx 提取章节结构（标题样式识别）+ 表格数据 + 叙述文字
- 9l.3.3 PDF 解析：MinerU GPU 加速（已有 Docker 配置）提取文字层 + OCR 扫描层/图片层
- 9l.3.4 LLM 智能结构化：解析后的原始文本交给 LLM 进行结构化处理
  - 识别章节边界（"五、1 货币资金"、"五、2 应收票据"等）
  - 提取表格数据（行列结构、数值）
  - 分离叙述文字与表格
  - 识别会计政策段落
- 9l.3.5 解析结果映射到当前模版结构：LLM 将历史附注的章节与当前模版章节自动对应
- 9l.3.6 上年数据自动填入"上期金额"列：历史附注中的期末余额 = 当年附注的期初/上期金额
- 9l.3.7 叙述文字预填：历史附注的会计政策、或有事项等叙述文字预填到当年附注，用户在此基础上修改

##### 9l.4 LLM 辅助附注编辑
- 9l.4.1 会计政策自动生成：根据模版类型（国企/上市）+ 行业特征 + 历史附注，LLM 生成标准会计政策文本
- 9l.4.2 变动分析自动生成：对比本期与上期数据，LLM 自动生成变动原因说明（如"应收账款较上年增长 30%，主要系本期销售规模扩大所致"）
- 9l.4.3 披露完整性检查：LLM 检查附注是否遗漏必要披露事项（如关联方交易、或有事项、日后事项）
- 9l.4.4 表述规范性检查：LLM 检查叙述文字是否符合会计准则表述规范（用语准确、逻辑通顺）
- 9l.4.5 智能补全：用户编辑叙述文字时，LLM 提供续写建议（类似 Copilot 体验）

##### 9l.5 附注导出为 Word — 标准化格式
- 9l.5.1 导出使用 python-docx 生成 Word 文档（不是 HTML 转 Word）
- 9l.5.2 排版规范：仿宋_GB2312 正文 + Arial Narrow 数字、页边距 3/3.18/3.2/2.54cm、表格上下 1 磅边框无左右
- 9l.5.3 章节标题样式：黑体加粗，按"一、二、三"/"（一）（二）"/"1. 2. 3."三级编号
- 9l.5.4 表格样式：三线表（表头下粗线+表尾粗线+中间细线），数字右对齐，合计行加粗
- 9l.5.5 页码：页脚居中，"第 X 页 共 Y 页"格式
- 9l.5.6 目录：自动生成附注目录（章节编号+标题+页码）
- 9l.5.7 交叉引用：报表行次旁的附注编号在 Word 中生成为超链接
- 9l.5.8 合并附注导出：包含合并特有章节，章节编号连续

##### 9l.6 附注文字编辑区域覆盖
- 9l.6.1 除科目主要注释（表格数据）外，附注还包含大量文字编辑区域：
  - 一、公司基本情况（企业概况、注册地、经营范围、母公司信息）
  - 二、财务报表的编制基础（持续经营假设、编制基础说明）
  - 三、重要会计政策和会计估计（20+ 个子章节：存货计价、固定资产折旧、收入确认、金融工具分类等）
  - 四、税项（增值税率、企业所得税率、其他税种及税率）
  - 六、关联方关系及交易（关联方清单、交易明细、应收应付关联方款项）
  - 七、或有事项（未决诉讼、对外担保、其他或有事项）
  - 八、承诺事项（资本承诺、经营租赁承诺）
  - 九、资产负债表日后事项（调整事项、非调整事项）
  - 十、其他重要事项（分部信息、持续经营、公允价值计量）
- 9l.6.2 每个文字区域支持 TipTap 富文本编辑 + LLM 辅助生成/润色
- 9l.6.3 文字区域可从历史附注预填，用户在此基础上修改更新
- 9l.6.4 会计政策章节按模版预填标准文本（国企版/上市版不同），用户可逐条修改
