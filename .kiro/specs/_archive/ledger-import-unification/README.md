# 账表导入统一方案（Ledger Import Unification）

## 背景

当前 `smart_import_engine.py` 已具备多文件多 sheet 智能识别能力（2800 行），但用户反馈有几类痛点：

1. 不同财务软件（用友、金蝶、SAP、Oracle、浪潮、Kingdee K3、新中大…）导出的余额表/序时账列名、结构差异大，偶发识别失败
2. 辅助余额表和辅助序时账**常常没有独立 sheet**，而是混在余额表/序时账里的"核算项目"列，当前拆分不稳定
3. 大文件（>100MB 的序时账）容易触发内存保护阈值或前端超时
4. 多 sheet 识别逻辑黑盒化，失败时错误信息不足以让用户定位到"哪个 sheet 的哪一列缺什么"
5. 错误未以弹窗结构化呈现，用户看不出"是文件问题还是系统问题、如何自救"

本 spec 目标：**建立一套覆盖"探测→识别→确认→解析→入库→反馈"全链路的通用账表导入规则**，既适配主流财务软件的导出格式，又允许用户在识别失败时介入修正。

## 核心原则

- **两阶段**：先只读前 20 行做快速探测（秒级返回识别结果），用户确认后再流式全量入库（异步后台）
- **三级识别**：Sheet 名 → 表头字段特征 → 人工确认，失败自动降级
- **四表联动**：余额表 / 序时账 / 辅助余额表 / 辅助序时账，缺任意一张从其他表虚拟派生
- **错误透明**：所有失败分"致命/阻塞/警告"三级，前端弹窗明确告诉用户"是什么问题 + 怎么修"
- **数据原子性**：staged 导入 + 显式 activate，中途失败不污染生产数据
- **关键列硬、非关键列软（核心）**：识别只在"四表联动依赖的关键列"上严格要求（置信度 ≥ 80 才算命中，否则强制人工确认）；次关键列（名称/摘要/制单人等）≥ 50 即自动映射，失败降级为 warning + 值置 NULL；非关键列（用户文件里我们没映射的列）原样入 `raw_extra JSONB`，不校验、不阻断。这样用户文件被"保留而非丢弃"，识别和校验聚焦在真正影响联动的列
- **通用规则、动态适配（v2.1 补充）**：识别规则声明式配置（JSON），表头+内容联合打分（并行而非串行降级），合并表头基于通用算法（值多样性）而非硬编码阈值，新增财务软件只加配置不改代码

## 关键里程碑

- **Sprint 0（第 0 周，外部依赖）**：样本收集 + 脱敏 + 反向工程，输出《财务软件导出格式对照表》
- **M1（第 1-2 周）**：`ledger_detector` 识别模块 + 8 家适配器骨架 + 单测覆盖 4 张表 3 级识别
- **M2（第 3 周）**：解析/校验/入库全链路可命令行跑通 1GB 序时账
- **M3（第 4-5 周）**：前端预检弹窗 + 列映射确认 UI + AI 兜底（可选）
- **M4（第 6-7 周）**：真实测试集（8 家财务软件样本）UAT 全通过，识别率 ≥ 85%

## 与既有系统的关系

- **复用**：`smart_import_engine.py` 的分批写入 / dataset 版本机制 / event_bus 激活事件
- **替换**：现有的 `_resolve_sheet_mapping` + `_build_sheet_context` 组合逻辑（全量重写为可测试、带置信度）
- **扩展**：新增 `LedgerDetectionResult` schema，前端预检 API 返回结构化探测结果

## 成功标准（QA）

1. 8 家财务软件导出样本，自动识别率 ≥ 85%，人工确认 1 列即可补齐的比例 ≥ 95%
2. 1GB 序时账（约 800 万行）导入耗时 < 10 分钟，峰值内存 < 2GB
3. 识别失败时，用户从错误弹窗能直接定位到具体 sheet 和列，无需看后台日志

## 灰度发布与回滚预案

**开关机制**：`feature_flags.py._DEFAULT_FLAGS["ledger_import_v2"] = False`（详见 design.md §13）

**Phase 0（内测 1 周）**：默认 False，指定 1-2 个内部测试项目 override 为 True

**Phase 1（Beta 2 周）**：允许业务方在项目设置里启用，自愿加入

**Phase 2（默认开启 2 周）**：默认 True，新项目走 v2，老项目手动迁移

**Phase 3（GA）**：无法关闭，全量

**Phase 4（清理）**：Phase 3 稳定 1 个月后删除旧 `smart_import_engine.py`

**回退条件**：新引擎报错率 > 5% 或 P95 耗时翻倍 → `feature_flags.set_project_flag(pid, "ledger_import_v2", False)` 单项目回退；大面积问题 → 改回 `_DEFAULT_FLAGS` 为 False 并热重启
