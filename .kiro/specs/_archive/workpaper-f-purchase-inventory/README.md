## 一、为什么做（基于真实模板核验的痛点）

合伙人/审计助理实际进入 F 采购存货循环底稿时遇到 **7 类核心问题**。F 循环是**成本审计的核心**，涉及 **F0 函证 / F1 预付账款 / F2 存货及跌价准备 / F3 应付票据 / F4 应付账款 / F5 营业成本** 共 6 个并列主底稿，其中 F2 存货是全系统**最复杂的单一底稿**（11 个物理文件 90 sheet，超过 D4 营业收入的 8 文件 48 sheet）。

### 1.6 F5 营业成本与 D4 营业收入 + F2 存货的"三角勾稽"缺失

**审计逻辑**：
```
营业收入(D4) - 营业成本(F5) = 毛利
存货期初(F2) + 本期采购 - 存货期末(F2) = 营业成本(F5)
```

**当前状态**：
- D4 有 VR-D4-01~04 勾稽规则（D spec 已实施）
- F5 与 D4 仅有 CW-153 单向引用
- F2 与 F5 之间**完全无勾稽规则**
- 三角关系（D4↔F5↔F2）未闭环

**需新增 VR 规则**：
- VR-F5-01：营业成本 = 期初存货 + 本期采购 - 期末存货（blocking）
- VR-F5-02：毛利率 = (D4 收入 - F5 成本) / D4 收入，波动 < 5%（warning，与 VR-D4-03 交叉验证）
- VR-F2-01：存货跌价准备计提率 vs 上年变动合理性（warning）
- VR-F2-02：存货周转天数 vs 行业均值合理性（warning）

### 10.3 与 prefill 的关系

- **Univer 内置 487 公式**：加载 xlsx 时自带，无需后端 prefill
- **prefill 17 cell**（当前）：为 F2-1 提供 TB 取数源（6 子科目 × 期初期末 + AJE/RJE/上年）
- **真正缺口**：F2-2 明细汇总表本身是空的（需要 =AUX 按子科目取数），F2-1 的 cross_sheet 公式才能有值
- **结论**：F2 的 prefill 扩展重点不在 F2-1（已有 487 内置公式），而在 **F2-2 明细汇总表**（=AUX 按子科目/仓库/产品取数）

1. `G1A-修订前`（F1 文件，旧版编号 G1 已重编为 F1，但 sheet 名未清理）

**扩展建议**（Sprint 1 第一个 task）：
```python
def _should_skip_historical_sheet(name: str) -> bool:
    if name is None:
        return False
    s = str(name)
    return (
        ("修订前" in s) or ("（原）" in s) or ("(原)" in s)
        or ("-删除" in s) or ("删除" in s and s.startswith("G"))
        or ("-移至" in s) or ("移至" in s and s.startswith("G"))
        or ("（示例）" in s) or ("(示例)" in s) or s.endswith("示例）")
    )
```
