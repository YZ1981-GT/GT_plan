- 真实数据验证：每个改动用 4 个项目验证
- 前端友好：每步有明确反馈
- 可扩展：公式/映射/模板都配置化

## §2 功能需求详述

### §2.A 报表引擎核心修复（F1-F2, F5-F6）

**F1 报表全零根因分析**：
- 已确认 trial_balance 有数据（陕西华氏 31 个非零科目）
- 已确认 formula 已填充（316 行，BS 55 行）
- 已确认 applicable_standard resolve 正确（soe_standalone）
- **最可能是测试脚本误报**（字段名 current_period_amount 而非 amount）
- 验收：陕西华氏 BS 至少 20 行非零

**F5 公式 seed 端点**：
- POST /api/report-config/fill-formulas 幂等执行
- 内部逻辑：按 row_name 匹配 → 特殊公式表 → CAS 标准 → 合计行自动生成
- 已有 scripts/fill_report_formulas.py 可复用核心逻辑

**F6 六种报表公式策略**：
- BS：TB('code','期末余额')，55/129 行覆盖（标题行/特殊行业行无公式正常）
- IS：TB('code','本期发生额')，16/78 行覆盖
- CFS：大部分 null（需序时账分析），仅小计行有 SUM_ROW
- EQ：TB('code','期末余额') 期末余额列
- CFS附表：间接法（净利润 + 调整项），ROW() 引用 IS 行次
- 减值准备：TB('code','期末余额') - TB('code','年初余额')

### §2.B 数据质量校验（F7-F8）

**检查公式**：期末余额 = 期初余额 + 本期借方发生额 - 本期贷方发生额
**容差**：±1 元
**差异分级**：blocking（>重要性水平）/ warning（≤重要性水平）/ info（≤1元）
**陕西华氏预期**：明细账不完整，应检出差异科目——这是正确行为，不是 bug

### §2.C 流程引导（F18-F19）

进度推导规则（从数据层，不用 localStorage）：
1. 导入完成：tb_balance > 0
2. 映射完成：account_mapping completion_rate >= 80%
3. 试算表完成：trial_balance > 0
4. 报表完成：financial_report > 0
5. 底稿完成：working_papers > 0
6. 附注完成：disclosure_notes > 0

### §2.D 前置条件校验（F26）

每个生成操作前自动检查依赖数据是否就绪，不满足时返回明确错误而非静默空结果。

**依赖链路**：
```
导入(tb_balance) → 映射(account_mapping) → 重算(trial_balance) → 报表(financial_report) → 底稿/附注
```

| 操作 | 前置条件 | 不满足时的错误信息 |
|------|---------|-------------------|
| recalc | account_mapping 存在且 rate≥50% | "请先完成科目映射（当前映射率 X%）" |
| generate_reports | trial_balance > 0 行 | "请先执行试算表重算" |
| generate_workpapers | template_set 已选择 | "请先选择底稿模板集" |
| generate_notes | financial_report > 0 行 | "请先生成财务报表" |

验收：每个操作前置条件不满足时返回 HTTP 400 + 明确中文错误信息 + `prerequisite_action` 字段指引下一步。

### §2.F 报表完整性说明（F28）

向用户解释为什么部分行没有数据：

| 类别 | 占比 | 原因 | 前端展示 |
|------|------|------|---------|
| 有公式+有数据 | ~25% | 正常取数 | 正常显示金额 |
| 有公式+数据为 0 | ~17% | 该科目余额确实为 0 | 灰显 "0.00" |
| 无公式（标题行） | ~20% | 分组标题，无需取数 | 加粗标题样式 |
| 无公式（特殊行业） | ~15% | 金融/保险行业专用，普通企业为空 | 斜体灰显 "—" |
| 无公式（CFS 明细） | ~15% | 需从序时账分析，非余额表可推算 | 显示"待填列" tag |
| 无公式（合计行缺失） | ~8% | 应有 SUM_ROW 但未生成 | 待修复 |

前端报表页面底部显示"数据覆盖率"摘要：
"本期报表 129 行，55 行有数据（42.6%），20 行标题行，35 行待手工填列，19 行特殊行业（不适用）"

### §2.G 数据质量检查套件（F29）

扩展 F7 为多种检查可选执行：

| 检查项 | 公式 | 级别 | 自动/手动 |
|--------|------|------|----------|
| 借贷平衡 | SUM(所有科目期末借方) = SUM(所有科目期末贷方) | P0 | 自动 |
| 余额表 vs 序时账 | 期末 = 期初 + 借方发生 - 贷方发生 | P1 | 自动 |
| 科目映射完整性 | 所有 tb_balance 科目都有 account_mapping | P1 | 自动 |
| 报表平衡 | BS 资产合计 = 负债合计 + 权益合计 | P1 | 报表生成后 |
| 利润表勾稽 | IS 净利润 ≈ 期末未分配利润 - 期初未分配利润 | P2 | 报表生成后 |

前端用红/黄/绿三色标识：
- 🔴 blocking：必须修复才能继续（如借贷不平衡）
- 🟡 warning：建议关注但不阻断（如余额差异 < 重要性水平）
- 🟢 passed：检查通过
