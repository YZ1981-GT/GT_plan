# 设计文档：底稿前端体验优化（wp-frontend-ux-polish）

## 概述

侧栏 13 Tab→4 功能组 + 8 横幅→折叠状态条 + 首次渐进引导 + HTML 化扩展 + 门禁引导式 + 异步进度统一。纯前端体验提升，不改业务逻辑。

## 核心设计

### 侧栏 4 功能组
| 组 | 包含 Tab | badge 汇总 |
|---|---|---|
| 编制辅助 | AI / 程序 / 程序要求 / 提示 | 高频 |
| 质量检查 | 自检 / 一致性 / 公式 / 复核标记 | 汇总未处理数 |
| 追溯关联 | 附件 / 依赖 / 证据 / 批注 | 中频 |
| 历史版本 | 版本 | 低频 |

实现：el-tabs 嵌套（一级 4 组 + 组内二级），保留所有 badge 上浮到组标签。

### 横幅折叠
- 阻断性（归档/编辑锁）：常驻显眼
- 信息性（AI/冲突/信任度/状态机/stale）：折叠为一行"⚠ N 项待处理" + 展开

### 渐进引导
- el-tour（Element Plus 2.4+）首次打开底稿编辑器时 spotlight 引导
- localStorage 记录"已引导"，可跳过/不再提示

### 门禁引导式
- `usePrerequisiteStatus` composable：进页面即预览"距离可生成还差 X 项"
- 替代现有"点了生成才报错"的拦截式

### 异步进度统一
- `AsyncJobProgress.vue`：统一进度条组件
- 所有异步任务（import/word-export/archive/generate）复用

## 不在范围
- 不改各子面板内部功能逻辑
- 不删任何现有 tab/功能（收纳≠删除）
