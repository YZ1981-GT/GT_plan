# 附注浮动行表格支持 — 需求文档

> 版本：v0.1（草稿）
> 编写：2026-05-28
> 范围：附注模板表格的「固定 / 浮动 / 固定+浮动」三态支持
> 前置：disclosure-note-full-revamp（已完成 46/47）

## 一、问题陈述

当前附注表格（`note.table_data.rows`）**全部按模板固定行数渲染**，但致同附注实际三种模式：

### 模式 A：固定行表格

行数与行标题完全由模板决定。例：

- 货币资金（库存现金 / 银行存款 / 其他货币资金 / 数字货币 / 合计）
- 资产负债表项目分类（项目 / 序号 / 本期金额 / 上期金额 — 行号固定）

### 模式 B：浮动行表格（行数与内容均动态）

行标题与数量都由实际业务数据决定。例：

- **应收账款 — 按欠款单位列示前五名**（5 行客户 + 1 行合计，但客户名是数据驱动的）
- **长期股权投资 — 子公司明细**（每个子公司一行，行数 = 子公司数量）
- **关联方关系 — 关联方清单**（每个关联方一行）
- **借款明细**（按贷款单位 × 借款种类一行）
- **存货分类构成**（按存货类别一行）
- **预计负债 — 按事项列示**（每项事项一行）

### 模式 C：固定+浮动表格（混合）

模板有固定的"骨架行"（如分类标题 / 合计 / 其中项），中间夹着浮动的明细行。例：

- **应付职工薪酬**：固定 6 类（短期薪酬 / 离职后福利 / 辞退福利 / 其他长期职工福利 / 内退人员费用 / 其中：高管薪酬），但每类下浮动子项
- **存货跌价准备变动**：固定列（期初 / 计提 / 转回 / 转销 / 期末），行是浮动存货类别
- **应交税费明细**：固定大类（应交增值税 / 企业所得税 / 个人所得税 / ...），但增值税下还有浮动子项（销项 / 进项 / 已交 / ...）
- **固定资产分类构成**：固定 7 类（房屋建筑物 / 机器设备 / 运输工具 / 电子设备 / 办公设备 / 酒店业家具 / 其他），其中"其他"下可能有浮动子项

### 现状缺陷

| 问题 | 影响 | 严重度 |
|------|------|------|
| **R1.1** 浮动行无法生成 | 173 章节中约 60+ 章节实际需要浮动，目前只能填空 | P0 |
| **R1.2** 用户在前端表样编辑器手工加行后无法持久化为模板 | 自定义模板数据 vs 模板定义混淆 | P0 |
| **R1.3** Word 导出按模板渲染，浮动加的行可能被覆盖 | 重新生成附注会丢用户数据 | P0 |
| **R1.4** 公式 binding 无法绑定到浮动行 | `=SUM(各浮动行)` 算不出 | P1 |
| **R1.5** 当前 row_type 只有 4 类（data/subtotal/total/header_label），缺 floating / placeholder 标签 | 无法区分模板空位 vs 用户加的行 | P0 |

## 二、用户需求（按角色）

### 审计助理（数据录入）

- **U1**：选中浮动表格区域 → 「+ 添加明细行」按钮一键加行（如新增一个客户/借款合同/存货类别）
- **U2**：在浮动行内，行标题（label 列）可手工输入，金额列既可手工填也可绑定取数公式
- **U3**：删除浮动行时弹确认，**不影响固定骨架行**
- **U4**：按"金额降序"排序浮动行（自动重排前 N 名）

### 项目经理（复核）

- **U5**：浮动行与固定行视觉区分（颜色/标记），便于审计师快速识别"哪些数据是助理填的、哪些是模板定义的"
- **U6**：合计行公式自动跨浮动行+固定行求和（`SUM(浮动区起 : 浮动区止)` 自动扩展）
- **U7**：上年附注的浮动行可"一键比对"导入（保留客户名 + 重新取数当年金额）

### 合伙人（最终签字）

- **U8**：附注 Word 导出后，浮动行使用与固定行**相同的致同三线表样式**，无视觉差异
- **U9**：浮动行段落用「（不适用的项目请删除）」标识与原模板一致
- **U10**：审计师对浮动行的修改有版本历史可回溯

## 三、验收标准

### A. 数据模型层（5 项）

- A.1 row 字段新增 `is_floating: bool`（默认 false）+ `floating_group: str`（同组浮动行属同一区域）
- A.2 row 字段新增 `is_placeholder: bool`（如模板里的「①」「……」标记）
- A.3 table_data 顶层新增 `_floating_regions: list[{name, start_idx, end_idx, expandable: bool}]` 描述浮动区
- A.4 binding JSON 支持 `floating_source: aux_balance | wp_data | manual`（浮动行的数据来源）
- A.5 现有 173 章节 binding 不破坏（向后兼容，缺 floating 字段视为全固定）

### B. 后端引擎（8 项）

- B.1 `DisclosureEngine._build_table_data` 识别浮动区，按 `floating_source` 实时展开行
- B.2 `aux_balance` 数据源支持按 aux_code 自动 explode（每个辅助账码一行）
- B.3 浮动行的 `label` 自动填充辅助账码名称（不是手工输入）
- B.4 浮动区合计行公式自动适配（`SUM(R3:R{N+2})` 中 N 由浮动行数决定）
- B.5 `update_note_values` 接收用户新增/删除/排序浮动行，merge 到既有 table_data 不丢失
- B.6 三态合并（`note_cell_merge.py`）兼容浮动行的新增/删除（不止 label 对齐）
- B.7 Sprint 3 自定义模板存储「用户加的浮动行」为 sidecar，不污染主模板
- B.8 PRIOR 函数支持跨年浮动行匹配（按 label 模糊匹配 + 数值取上年）

### C. 前端编辑器（6 项）

- C.1 表样编辑器渲染时浮动区背景浅黄底色（视觉区分）
- C.2 浮动区第一行上方显示「+ 添加明细行」按钮（仅浮动区内出现）
- C.3 删除浮动行右键菜单，固定行右键菜单**不显示删除**项
- C.4 浮动行行号标记 `★`（区分模板固定行 `R1/R2` 与浮动行 `R*1/R*2`）
- C.5 选中浮动行 → 公式栏显示「该行可绑定 aux_balance / 手工」选项
- C.6 浮动区版本历史按"加行/删行/改值"事件分别记录

### D. Word 导出（4 项）

- D.1 浮动行使用 GTNoteFloatingRow 段落样式（与 GTNoteTableRow 字号缩进一致，仅内部 metadata 区分）
- D.2 浮动区结尾自动加「（不适用的项目请删除）」灰色提示文字（仅 SOE 模板）
- D.3 浮动行的合计行使用粗下边框（与固定合计行一致）
- D.4 视觉断言新增 4 项：浮动行字号 / 浮动行行高 / 浮动区底色（导出灰白） / 不适用提示文字字号

### E. 兼容与回退（3 项）

- E.1 新模板字段（is_floating / _floating_regions）**仅当存在时启用**，否则按现有固定逻辑跑
- E.2 既有 4101 binding cells 不动，仅新增浮动 binding 单独存储
- E.3 一键回退：把 `_floating_regions = []` 即退回纯固定模式

## 四、范围与不做事项

### 必做

- 173 章节中**已识别的 60+ 浮动表**列入 binding（应收账款前五名 / 借款明细 / 子公司明细 / 关联方清单 / 存货分类 / 应交税费明细 / 固定资产分类 / 应付职工薪酬等）
- 前端表样编辑器加行按钮 + 浮动区视觉
- Word 导出浮动行样式

### 不做（v1）

- 浮动列（横向浮动）— 暂不支持
- 跨章节浮动行联动（如 A 章节加客户，B 章节自动同步）
- AI 自动识别哪些行该浮动（依赖审计师手工标记 binding）

## 五、性能 / 数据量预估

- 60+ 浮动表 × 平均 5-20 浮动行 = 总浮动 cells 约 600-1500
- 浮动行展开 = `aux_balance` 查询，单次 < 100ms（已有索引）
- 全量 173 章节生成耗时增加不超过现有的 30%（基线约 8s → 上限 11s）
