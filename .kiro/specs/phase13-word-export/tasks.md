## 9. 回滚预案

| 场景 | 回滚动作 | 恢复目标 | 负责人 |
|------|---------|---------|--------|
| 模板填充生成异常 | 关闭 Word 导出入口，保留既有 PDF/Excel 导出路径 | <10分钟 | 开发负责人 |
| 导出任务状态异常 | 回滚 `export_jobs/export_task` 相关迁移并恢复旧导出入口 | <20分钟 | 后端负责人 |
| 历史补录异常 | 执行补录回滚脚本并恢复旧模板类型口径 | <30分钟 | DBA |
| ONLYOFFICE/字体故障 | 切换为方案A降级导出或仅保留下载已生成版本 | <10分钟 | 运维 |

## 10. 风险与应对

| 风险 | 应对 |
|------|------|
| 模板与致同标准版式不一致 | 灰度前必须完成模板对版清单并逐套签收 |
| 快照过期导致导出旧数据 | stale 检测 + 显式重算或确认沿用旧快照，不允许静默继续 |
| ONLYOFFICE/模板/字体异常 | 预检告警 + 方案A降级导出，不阻断下载 |
| 全套导出长任务中断 | 基于 `job_id` 恢复进度，仅重试失败项 |
| 自定义模板引用失效 | 读取项目级 `custom_template_snapshot`，缺失时阻断并给出业务错误 |
| 上年 PDF 报告 OCR 质量差 | MinerU 解析 + 质量检测 + 人工确认 |
| LLM 预填内容不准确 | 所有 LLM 内容标记 `[需确认]`，人工逐段/逐章确认 |
| 集团子企业会计政策差异大 | 差异标记+绿色高亮提示，不自动替换政策段落 |
| 附注 manual 数据被自动刷新覆盖 | manual 优先级最高，刷新仅更新 auto 单元格 |
