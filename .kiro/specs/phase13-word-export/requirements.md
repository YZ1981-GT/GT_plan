# Phase 13: 致同标准 Word 导出引擎 - 需求文档

> 审计报告、财务报表、附注三类正式文档的 Word 导出，严格遵循致同排版规范。

---

## 1. 背景与目标

### 1.1 现状

| 文档类型 | 当前能力 | 问题 |
|---------|---------|------|
| 审计报告 | 仅 HTML/PDF（pdf_export_engine.py 简单模板） | 无 Word 导出，无致同标准排版，无签章页 |
| 财务报表 | 仅 HTML/PDF（_render_financial_report_html） | 无 Word 导出，无三线表，无缩进层级 |
| 附注 | 有基础 Word（note_word_exporter.py） | 缺三线表、页眉页脚、三色文本处理、千分位 |

核心问题：审计报告是给客户和监管机构看的正式文件，格式必须精确到毫米级。HTML→PDF 方案无法满足致同排版标准。

### 1.2 核心目标

| 目标 | 量化指标 |
|------|---------|
| 格式精确度 | 与致同标准 Word 模板 100% 一致 |
| 导出速度 | 单文档 <10秒，全套（报告+报表+附注）<30秒 |
| 用户可编辑 | 导出后可通过 ONLYOFFICE 在线编辑微调 |
| 模板可更新 | 换模板文件即可，不改代码 |

### 1.3 双方案策略

| | 方案A: python-docx 从零生成 | 方案B: Word 模板填充+ONLYOFFICE 编辑 |
|---|---|---|
| 格式精确度 | 难以 100% 还原 | 100%（模板自带格式） |
| 开发工作量 | 大（每个细节都要编码） | 小（只写填充逻辑） |
| 用户可编辑 | 导出后用本地 Word 改 | 在线直接编辑 |
| ONLYOFFICE 依赖 | 不依赖 | 依赖（降级用方案A） |

**决策：方案B 优先，方案A 作为降级备选。**

## 3. 致同标准排版规范

### 3.1 页面设置
- 页边距：左 3cm、右 3.18cm、上 3.2cm、下 2.54cm
- 页眉 1.3cm、页脚 1.3cm

### 3.2 字体字号
- 中文（含页眉页脚）：仿宋_GB2312、小四（12pt）
- 数字和英文：Arial Narrow
- 表格金额过大时：最小 9pt

### 3.3 三色文本规则（附注专用）
- 黑色：正式披露内容（保留）
- 蓝色：编写者指引（导出时删除）
- 红色：披露格式提示（导出时转黑色）

### 3.4 文字排版
- 标题行：左缩进 -2 字符，首行不缩进，居左
- 段落间距：段前 0 行、段后 0.9 行，单倍行距
- 表格下一行：段前 0.5 行

### 3.5 三线表格式
- 上下边框 1 磅，标题行下边框 1/2 磅，无左右边框
- 标题列左对齐，数据列右对齐，垂直居中
- 标题行+合计行加粗
- 数字千分位格式（#,##0.00），负数用括号

### 3.6 页眉页脚
- 页眉左：事务所名称 | 页眉右：项目名称
- 页脚居中："第 X 页 共 Y 页"（python-docx OxmlElement 域代码）
