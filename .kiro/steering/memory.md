## 操作铁律（标题级，详见 #conventions）

- **彻底解决不绕开**：错误必须复现+根因+修主代码+防御测试
- **三层一致校验**：DB 迁移 + ORM `Mapped[]` + service 方法，任一缺失即伪绿
- **可复用脚本沉淀**：批量入库/UAT/迁移工具放 `backend/scripts/{name}.py` 配 docstring + 多场景
- **PG 运维**：SET 不支持绑定参数（用 set_config）/ superuser bypass RLS / CONCURRENTLY 必须 asyncpg raw conn + lock_timeout
- **router_registry 必查铁律**：新建 router 文件后必须在 `backend/app/router_registry/{group}.py` 注册，否则 endpoint 写好但前端 404
- **WpFileStatus 完成语义**：「底稿已完成」= `status in (review_passed, archived)`，不要猜不存在的枚举
- **临时文件不进 commit**：commit-msg.txt 必须 `git rm --cached` 清掉；优选 `git commit --% -m "..."` stop-parsing token
- **agent 调 service 优于 Playwright UI**：大文件入库直调 ledger_import 管线快 10x
- **历史档案不回填修改**：dev-history.md / spec/tasks.md 等是 append-only 审计轨迹，目录重组时不回填旧路径
- **vue-tsc 类型债务清理 SOP**：mitt Events 类型表必须显式补 key；SyncEventPayload 用 escape hatch；FUniver/xlsx SDK 用 `(api as any)` cast
- **getDiagnostics 假阳性 vs vue-tsc 权威性铁律**（2026-05-30 实证）：`.vue` SFC 被编辑后（如 GtAmountCell.vue 删 inject），IDE 语言服务（Volar/TS-server）的增量 `.vue.d.ts` 投影缓存会陈旧，对 import 该 SFC 的文件报假阳性 `找不到模块 @/.../X.vue`；**判定方法** = ①跑 `vue-tsc --noEmit`（编译器，权威）= 0 错则是假阳性 ②byte-identical 的同路径 import 在其他文件不报 ③报错在多个文件间游走/编辑 import 不消失 ④runtime vitest 通过；**根因** = TS server 缓存非代码缺陷，editCode 无法修复，需 IDE "Restart TS Server"/reload window 清缓存；**合法抑制手段** = `// @ts-expect-error — TS2307 false positive: Volar stale cache; vue-tsc 0 errors, runtime passes`（不用 `@ts-ignore`），好处是缓存修复后 `@ts-expect-error` 变 unused 自动报 warning 提醒移除；**禁止**伪造代码改动（inline `declare module` / 改 import 路径 / 加无用 reference）来消红线；与既有"vue-tsc 才报 getDiagnostics 不报"（element-plus 类型陷阱）方向相反
- **Vue setup const 声明顺序铁律**（2026-05-25）：`const X = useY(..., Z)` 引用的 Z 必须在 X 之前已定义；典型实例 = WorkpaperEditor commit 79f7936 把 6 个 cycle composable 实例化放在 cycleDialogs 之前触发 ReferenceError；判定 = const 链式依赖必须按拓扑顺序写
- **顶层 v-if 守卫拦 init 死锁铁律**：依赖 template ref 触发 init 的组件不能加顶层 `v-if="loading"` 守卫，改 overlay 模式（容器永远渲染 + 内部蒙层）
- **PowerShell Set-Content 破坏 UTF-8 铁律**（2026-05-29）：`(Get-Content ... -Raw) -replace ... | Set-Content` 会把含中文的 .py 文件编码截断为 `�`（BOM/编码丢失）；**禁止用 PowerShell 写含中文的源文件**；正确做法 = 用 `.venv\Scripts\python.exe -c "..."` 脚本做文本替换（显式 `encoding='utf-8'`）
- **.bat 禁用 Unicode 框线字符铁律**（2026-05-29）：`╔═╝╗║│└┘` 等 Unicode box-drawing 字符在 cmd 下会被解析为命令导致 `'╝' is not recognized as command`，即使 `chcp 65001` 也不稳定；正确做法 = 用 ASCII `+-|=` 替代；`|` 在 echo 中必须用 `^|` 转义
- **SQLAlchemy text() SQL 注释 :name 陷阱**（2026-05-29）：`text()` 即使在 `--` SQL 注释中也会扫描 `:name` 模式当成 bind parameter（V005__enable_rls.sql 注释里的 `:pid` 触发 `A value is required for bind parameter 'pid'`）；正确做法 = 注释中用 `$1`/`$N` 占位符（PG 原生 prepared statement 风格）；**根本解决方案**（migration-runner-resilience 落地）= `conn.exec_driver_sql(stmt)` 替代 `conn.execute(text(stmt))` 跳过 SQLAlchemy bind 解析，注释/字符串内 `:name` 全当字面量 — 仅系统 INSERT schema_version 一句保留 `text+bind`
- **Windows asyncio subprocess 不可用陷阱**（2026-05-29）：uvicorn `--reload` 启动的子进程在 Windows 上 `asyncio.create_subprocess_exec` 抛 `NotImplementedError`（ProactorEventLoop 限制）；正确做法 = `subprocess.run + loop.run_in_executor` 同步包装，或显式 `asyncio.set_event_loop_policy(WindowsSelectorEventLoopPolicy())`
- **DB 迁移幂等铁律补强**（2026-05-29）：所有 `CREATE TABLE/INDEX` 必须 `IF NOT EXISTS`，否则启动重跑会触发 `DuplicateTableError` 中断 D6 管线；V006/V010 实测漏加导致每次启动报错；**ALTER TABLE 也需防御**：表不存在时（如 `knowledge_documents` 由 ORM 启动时建）`ALTER TABLE` 会抛 `UndefinedTableError`，正确做法 = 用 `DO 'BEGIN IF NOT EXISTS (...) THEN RETURN; END IF; ALTER TABLE ...; END'` 包装（V016 案例）
- **LibreOffice 启动检查简化**（2026-05-29）：Windows 启动 soffice --version 需 8-15 秒（外加进程冷启动），不应在后端启动时阻塞；改为仅 `_find_soffice()` 路径检测（毫秒级），运行时按需启动 + 三级数据源前两级 fallback
- **冗余别名 const 直接路径化铁律**（2026-05-27）：`const X = SOURCE.path` 形式的局部别名（如 `stocktakeDialogVisible = cycleDialogs.stocktake.visible`）应直接在模板中用 `cycleDialogs.stocktake.visible.value` 路径访问；函数式 wrapper（`function onX(s) { SOURCE.method(s) }`）保留无妨，构成 `const onX = SOURCE.handlers.onX` 的纯转发别名必须删除
- **subagent 任务窗口拆分铁律**（2026-05-27）：超 2000 行 SFC 重构按依赖拓扑独立子任务（toolbar → mode → cycles → aliases），每子任务 vitest + vue-tsc 双卡点，可在静态环境完成 80% 工作 + Playwright 真实环境兜底 20%；典型实例 = WorkpaperEditor 12.1.1-12.1.5 五子任务串行（每个 -10~70 行）累计 -70 行 / 87 errors 持平 / 0 测试回归
- **PowerShell -replace 中文乱码铁律**（2026-05-28）：PowerShell `(Get-Content $f -Raw) -replace pattern, replacement | Set-Content` 处理含中文/emoji 的 .ts/.vue 文件会导致 UTF-8 → GBK 转换乱码（`表格` → `表�?`），**绝对禁止**；批量编辑必须用 fsWrite / strReplace / sed（git bash 下），失误了立即 `git checkout HEAD -- {file}` 回退
