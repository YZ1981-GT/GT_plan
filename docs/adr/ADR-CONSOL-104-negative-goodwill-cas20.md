# ADR-CONSOL-104: 负商誉按 CAS 20 全额计入当期损益（删除编造的递延摊销）

## 状态
已接受（待审计专业确认）(2026-05-31)

## 背景

`goodwill_service.calculate_goodwill` 的负商誉处理 `treatment = "计入损益" if abs(goodwill) < acquisition_cost * 0.25 else "递延收益摊销"` 是**编造的会计逻辑**——CAS 20《企业合并》规定负商誉经复核后全额计入当期损益（营业外收入），无"25% 阈值"、无"递延收益摊销"（后者是旧准则/早期 IFRS 做法，现行 CAS 已废止）。"实现了但算错"比"没做"更危险（用户会采信）。

## 决策

- 删除 25% 阈值 + 递延摊销分支。
- 负商誉统一 `treatment = "计入当期损益（营业外收入）；需复核合并成本与可辨认净资产公允价值的计量"`。
- 正商誉 `treatment = "确认为商誉"`。
- 商誉计算公式本身（成本 − 可辨认净资产 FV × 母持股比例）**不变**。

## 后果

- 会计处理符合现行准则。
- 代价：负商誉计量符合性最终须由懂 CAS 20 的审计专业人员复核后方可标完成（任务 6A.3 标 `[ ]* 待审计专业确认`，代码 review 看不出准则错误）。
- 测试 `test_goodwill.py` 更新：正商誉 treatment=="确认为商誉"，负商誉含"当期损益"且不含"递延"，大额负商誉也不再走递延摊销。
