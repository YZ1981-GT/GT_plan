# 审计底稿系统深度开发方案（重构版）

> **目标**：构建"AI草拟→人工确认→AI优化→人工定稿"的人机协同底稿编制体系，实现审计说明自动生成、复核效率提升30%、项目经理全局管控。

---

## 一、执行摘要

### 1.1 现状一句话
底稿系统核心能力（模板生成、WOPI编辑、QC自检、复核批注）已做实，但**审计说明依赖手工填写**（每张底稿200-500字，占编制时间50%+）、**AI能力未激活**（LLM调用无prompt工程）、**角色视图无差异**（四种角色看到相同界面）。

### 1.2 核心目标
| 目标 | 量化指标 | 实现方式 |
|------|---------|---------|
| 审计助理效率提升 | 单张底稿编制时间 -40% | AI生成审计说明草稿 |
| 项目经理复核效率提升 | 单张底稿复核时间 -30% | AI预审+复核工作台 |
| 数据一致性保障 | 底稿-试算表差异率 <1% | 自动校验+冲突提示 |
| 质量可控 | QC通过率从 85% → 95% | 内容级QC规则+AI辅助 |

### 1.3 预期收益
- **人效**：363张底稿项目，审计助理平均节省 3-5 人天
- **质量**：审计说明完整性从 60% → 90%，数据一致性错误归零
- **体验**：复核人无需等待锁即可浏览底稿，项目经理一眼看清全项目进度

### 1.4 范围边界（本期不做）

| 功能 | 原因 | 后续版本 |
|------|------|---------|
| 底稿版本对比（两版本差异） | 需存储多版本Excel文件，存储成本高 | P4阶段（延伸功能） |
| 移动端底稿编辑 | 屏幕尺寸限制，审计场景以PC为主 | 暂不考虑 |
| 自动审计程序执行 | 需对接客户ERP系统，接口不统一 | 专项迭代 |
| 语音输入审计说明 | 准确率不足，审计术语识别差 | 等待ASR模型成熟 |
| 实时协同编辑（多人同时编辑） | 冲突解决复杂，审计场景单编制人为主 | 远期规划 |

## 二、问题分层（P0-P2）

### P0 — 阻塞问题（立即修复，3天）

| 序号 | 问题 | 影响 | 修复方案 |
|------|------|------|---------|
| P0-1 | `prefill_stale=true` 前端无感知 | 用户不知道底稿数据已过期 | 三处提示：底稿列表橙色⚠图标、编辑器顶部黄色横幅、工作台树节点标记 |
| P0-2 | WOPI互斥锁阻挡复核人浏览 | 复核人想看底稿必须等编制人关闭 | check_file_info返回ReadOnly=True，复核人只读模式打开 |
| P0-3 | 审计说明不提取到parsed_data | 复核人看不到说明摘要，AI无法引用 | parse_workpaper_real增加说明提取（关键词搜索+连续文本） |
| P0-4 | 底稿列表无角色裁剪 | 审计助理看到全部300+底稿，无法聚焦 | 前端配合scope_cycles后端过滤，只显示负责循环 |

### P1 — 核心缺失（7天）

| 序号 | 问题 | 影响 | 方案 |
|------|------|------|------|
| P1-1 | 审计说明纯手工编写 | 最耗时环节，占编制时间50%+ | AI智能生成：数据采集→Prompt构建→流式输出→人工编辑→写入底稿 |
| P1-2 | 复核流程效率低 | 打开→浏览→写意见→通过/退回，逐个操作 | 复核工作台：待复核队列+AI预审+批量通过/退回 |
| P1-3 | 项目经理无全局视图 | 无法一眼看清全项目底稿完成情况 | 进度总览面板：矩阵+指标卡片+甘特图+人员负荷 |
| P1-4 | 底稿-试算表数据不一致 | 审定数与试算表差异无法及时发现 | 数据一致性监控：矩阵对比+自动校验+一键同步 |
| P1-5 | 批量操作缺失 | 分配/刷新/生成说明只能逐个操作 | 批量操作面板：checkbox选择+批量API |

### P2 — 体验优化（5天）

| 序号 | 问题 | 影响 | 方案 |
|------|------|------|------|
| P2-1 | QC规则粒度粗（仅14条，无内容检查） | 内容级问题无法自动拦截 | 新增4条内容级规则：说明完整性、数据一致性、附件充分性、交叉引用完整性 |
| P2-2 | 合伙人签字前无专项检查 | 签字前遗漏问题发现晚，返工成本高 | 签字前底稿检查清单：5项专项检查（复核状态/QC通过/说明非空/数据一致/证据充分） |
| P2-3 | 合伙人淹没在300+底稿中 | 高风险底稿被淹没，无法聚焦关键风险 | 风险底稿聚焦视图：自动筛选高风险科目（金额>重要性/有调整/QC阻断/被退回过） |
| P2-4 | 质控人员无法抽查 | 质控独立于项目团队，需独立抽查机制 | 独立抽查工作台：智能抽样+检查清单+抽查报告生成 |
| P2-5 | 附件与底稿关联不可见 | 证据链不清晰，复核人无法快速定位证据 | 证据链可视化：关联列表+快速关联+证据图谱 |

## 三、核心方案 — 人机协同工作流

### 3.1 设计理念

```
AI不替代审计判断，AI是审计助理的"副驾驶"
├── AI草拟 → 人工审阅确认 → AI根据反馈优化 → 人工最终定稿
├── 所有AI生成内容必须有明确标记（ai_content字段）
├── 未经人工确认的AI内容 → QC规则阻断提交复核
└── AI建议可一键采纳或忽略，不强制
```

### 3.2 全流程五阶段

```
┌─────────────────────────────────────────────────────────────┐
│  Phase 1: 底稿生成（全自动）                                   │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐              │
│  │ 模板复制  │ →  │ 表头填充  │ →  │ 公式预填  │              │
│  │ (自动)    │    │ (自动)    │    │ (自动)    │              │
│  └──────────┘    └──────────┘    └──────────┘              │
│  输出：带审定数公式的底稿草稿                                   │
├─────────────────────────────────────────────────────────────┤
│  Phase 2: 数据核对（人机协同）                                │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐              │
│  │ AI标记   │ →  │ 人工核对  │ →  │ 确认/调整 │              │
│  │ 异常值   │    │ 逐行确认  │    │ 修正数据  │              │
│  └──────────┘    └──────────┘    └──────────┘              │
│  异常值：审定数与试算表差异>阈值、变动率>30%、AJE金额>重要性水平  │
├─────────────────────────────────────────────────────────────┤
│  Phase 3: 审计说明（人机协同★核心创新）                       │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐              │
│  │ AI草拟   │ →  │ 人工编辑  │ →  │ AI优化   │              │
│  │ 说明草稿 │    │ 修改补充  │    │ 润色定稿  │              │
│  └──────────┘    └──────────┘    └──────────┘              │
│  ↓ 用户可随时切换为纯手动模式                                 │
│  数据采集：试算表+调整分录+科目映射+TSJ要点+抽样结果            │
│  Prompt：角色=资深审计员，格式=致同标准（目标→程序→发现→结论）│
├─────────────────────────────────────────────────────────────┤
│  Phase 4: 自检提交（自动+人工）                               │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐              │
│  │ QC自检   │ →  │ 人工修复  │ →  │ 提交复核  │              │
│  │ (14+4条) │    │ 阻断项    │    │ (门禁检查)│              │
│  └──────────┘    └──────────┘    └──────────┘              │
│  QC-02门禁：检查ai_content中status=pending的项               │
├─────────────────────────────────────────────────────────────┤
│  Phase 5: 复核（人机协同）                                    │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐              │
│  │ AI预审   │ →  │ 复核人    │ →  │ 通过/退回 │              │
│  │ 标记问题 │    │ 人工判断  │    │ (附原因)  │              │
│  └──────────┘    └──────────┘    └──────────┘              │
│  AI预审检查：数据一致性、说明完整性、结论合理性                 │
└─────────────────────────────────────────────────────────────┘
```

**Phase 4数据一致性校验细则**

| 校验项 | 校验逻辑 | 阻断/警告 |
|--------|---------|----------|
| 校验1 | 底稿审定数 vs 试算表审定数，误差>0.01元 | 阻断 |
| 校验2 | 底稿AJE合计 vs adjustments表AJE合计 | 阻断 |
| 校验3 | 底稿RJE合计 vs adjustments表RJE合计 | 阻断 |
| 校验4 | 交叉引用底稿状态，禁止引用draft状态底稿 | 阻断 |
| 校验5 | 审计说明字数<50字 | 警告 |
| 校验6 | 审计说明字数<20字 | 阻断 |
| 校验7 | 重要性以上科目（金额>materiality）附件数=0 | 阻断 |
| 校验8 | prefill_stale=true且超过24小时未刷新 | 警告 |
